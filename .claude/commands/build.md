`pnpm build`를 실행하고 결과를 간결하게 보고한다.

## 실행

Bash 툴로 `pnpm build` 실행. 출력이 길 수 있으니 timeout은 300000ms 정도로 넉넉히.

## 보고 원칙

- 성공 시: "빌드 성공" 한 줄 + 생성된 주요 산출물 경로 몇 줄 (`dist/` 하위) 또는 번들 크기 요약만.
- **폰트 subset 로그 (`Pretendard-*.subset.woff2` 같은 줄 90개 이상)는 나열하지 말 것.** 요약만.
- 실패 시: 에러 메시지와 파일 경로·라인 번호를 그대로 옮기고, 원인 추정/수정 제안을 짧게.
- tsc 에러와 vite 에러를 구분해서 표시.

## 빌드 후 테스트 체크리스트

빌드 성공 시, 이번 세션에서 변경된 파일을 `git diff` 또는 `git log`로 확인하고, 변경 내용에 해당하는 테스트 항목을 체크리스트로 출력한다.

- 변경된 컴포넌트/로직별로 사용자가 Chrome에서 직접 확인해야 할 항목을 `- [ ]` 형식으로 나열.
- UI 변경: 해당 화면을 열어서 눈으로 확인할 수 있는 항목 위주.
- 로직 변경: 해당 플로우를 실행해볼 수 있는 시나리오 위주.
- 변경이 없거나 문서만 바뀐 경우 체크리스트 생략.

## 주의

- 빌드 전에 불필요한 파일 수정 금지. 사용자가 "수정하고 빌드"라고 명시하지 않은 이상 이 스킬은 **빌드만** 한다.
- 빌드 후 `dist/`의 매니페스트 재로드는 사용자가 Chrome에서 직접 해야 한다는 점을 필요시 상기.
