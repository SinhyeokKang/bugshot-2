# source-command-guide

Use this skill when the user asks to run the migrated source command `guide`.

## Command Template

`guide/ko`·`guide/en` 사용 가이드를 **작성·갱신**하는 전용 스킬. 코드(`src/`)는 건드리지 않고 `guide/`만 다룬다. 작성 규칙·IA·톤·사실 대조 소스는 전부 **`guide/AUTHORING.md`가 단일 출처** — 이 스킬은 그 매뉴얼을 로드해 실행하는 손이다.

핵심 분리: `/push`는 가이드 stale을 **감지·경고**하는 가벼운 게이트, `/guide`는 **실제 작성·갱신**을 자기완결한다. 사용자 노출 기능을 바꾼 `/implement` 보고의 "가이드 영향 ⚠️" 플래그가 이 스킬의 진입 신호다.

## 사용

- `/guide` — 직전 컨텍스트(방금 구현한 기능, `/implement`의 "가이드 영향" 플래그)에서 갱신 대상 추론.
- `/guide <slug 또는 설명>` — 대상 명시. 예: `/guide ai-styling`, `/guide 30초 리플레이 권한 안내`.
- `/guide sync` — 전체 대조. 코드 현재 상태와 guide/ 23페이지를 전수 비교해 stale 페이지를 모두 찾아 갱신.
- `/guide new <섹션>` — 새 페이지·섹션 추가(IA 변경) — SUMMARY·트리·footer까지.

## 다른 스킬과의 분리

- `/implement` — 코드 구현. 가이드는 "영향 플래그"만 남기고 작성 안 함.
- `/guide` ← 여기. **guide/ko·en 작성·갱신 + 검증**. 빌드·커밋 안 함.
- `/push` — 가이드 stale 감지 + 신선도 게이트. 규모 작으면 직접, 크면 `/guide` 권장.

## 절차

### 0. AUTHORING.md 로드 (필수 선행)

**가장 먼저 `guide/AUTHORING.md`를 읽는다.** 운영 방식·IA·이슈 작성 공통 흐름·사실 대조 소스·톤앤매너·UI 라벨 규칙·마크다운/footer 규칙·ko/en 대칭·검증 체크리스트가 전부 거기 있다. 이 단계를 건너뛰고 작성하지 않는다.

### 1. 대상 결정

- 인자 또는 직전 컨텍스트에서 갱신 대상 페이지를 확정한다.
  - `/implement` 직후면 그 보고의 "가이드 영향" 대상 페이지를 채택.
  - `sync`면 23페이지 전체가 대상.
  - 자연어면 영향 페이지를 AUTHORING.md의 IA 트리에 매핑.
- 멀티 페이지면 TaskCreate로 추적.

### 2. 코드 대조 (사실 확정)

AUTHORING.md "사실 대조 소스" 표의 경로를 읽어 **현재 코드의 사실**을 확정한다 — 단축키·본문 섹션 기본값·라벨(i18n ko/en)·스타일 섹션 순서·로그 정책·마커·플랫폼 표 등. **추측 금지** — 가이드 문구가 코드와 어긋나면 안 된다. 변경된 기능이면 그 기능의 i18n 키·동작을 직접 확인한다.

### 3. 작성·갱신 (메인 스레드 단일, ko·en 동시)

- AUTHORING.md 규칙대로 작성한다:
  - **ko·en을 그 자리에서 동시에** 갱신(분리 금지 — 드리프트 방지).
  - **톤**: ko = "-습니다" 유지 + 캐주얼·쿠션어 / en = 친절·격려, 직역 금지.
  - **UI 라벨**: 로케일별 실제 화면 라벨 인용(영문 식별자 금지). 캡처 모드·본문 섹션·버튼명은 i18n 실문구.
  - **이미지**: 더미 경로 깊이(루트 `assets/`, 하위 `../assets/`) + 의미 있는 캡션.
  - **footer**: 본문 페이지 맨 아래 언어 전환 링크 1개(ko→en홈 / en→ko홈). SUMMARY 제외.
  - **비표준 확장 문법(`{% hint %}`) 미사용** — 주의는 plain 인용구(`>`).
- IA가 바뀌면(페이지 추가/삭제) ko·en `SUMMARY.md`와 트리를 양쪽 대칭으로 갱신.
- **외과적**: 대상 페이지만. 무관한 페이지 톤·문구 임의 변경 금지.

### 4. AUTHORING.md 동기 점검

이번 변경이 **가이드 작성 기준 자체**를 바꿨으면(새 플랫폼 → 플랫폼 표, 단축키/로그 정책/본문 섹션 변경 → 사실 스냅샷, 새 페이지 → IA 트리) `guide/AUTHORING.md`의 해당 표·스냅샷도 함께 갱신한다.

### 5. 검증 (AUTHORING.md 체크리스트)

AUTHORING.md "검증 체크리스트"의 정적 점검을 실행한다(`cd guide`):
- 플레이스홀더 0건 / ko·en 파일 트리 대칭 / 페이지별 헤딩 수 대칭 / 내부 `.md` 링크 무결 / 더미 이미지 경로 깊이 유효 / footer 페이지당 1개.

순수 문서라 `pnpm test`·`pnpm build`는 돌리지 않는다.

### 6. 보고 + 종료

```
대상: <페이지 N개 / sync / slug>
갱신 파일: <guide/ko/…, guide/en/… 목록>
사실 대조: <확인한 코드 소스 — 라벨/단축키/정책 일치 여부>
AUTHORING.md 동기: <갱신함(표/스냅샷) / 변경 없음>
검증: 플레이스홀더 0 · ko/en 트리·헤딩 대칭 · 링크·이미지·footer OK
```

- 보고 후 **종료**. 빌드·커밋·푸시 안 함(커밋은 `/push`·`/merge`가 `docs(guide): ...`로).

## 금지 사항

- **AUTHORING.md 안 읽고 시작 금지** — 0단계는 필수.
- **ko/en 분리 작성 금지** — 한 페이지는 그 자리에서 양 언어 동시.
- **코드 수정 금지** — `guide/`만. `src/`·manifest·i18n은 건드리지 않는다(읽기만).
- **사실 추측 금지** — 라벨·단축키·정책은 코드 재확인.
- **빌드·커밋·푸시 금지**.
- **범위 밖 페이지 임의 워싱 금지** — 대상 페이지만.
