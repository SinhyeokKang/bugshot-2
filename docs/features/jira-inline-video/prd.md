# Jira 인라인 비디오 임베드

## 배경

현재 화면 녹화(`recording.webm`)는 Jira 이슈에 첨부파일로만 업로드된다. ADF 본문에는 "(recording.webm 참조)"라는 텍스트만 들어가서, 이슈를 열었을 때 비디오를 바로 재생할 수 없고 첨부 목록에서 별도로 클릭해야 한다.

스크린샷은 이미 ADF `mediaSingle > media` 노드로 본문에 인라인 삽입되어 Jira에서 바로 보인다. 비디오도 동일한 패턴으로 삽입하면 이슈 본문에서 바로 재생 가능하다.

추가로, Jira의 미디어 플레이어는 MP4(H.264) 지원이 WebM보다 안정적이므로, 제출 시점에 WebM → MP4 변환을 수행한다.

## 목표

1. Jira 이슈 본문(ADF)에 녹화 비디오가 인라인으로 임베드되어, Jira에서 이슈를 열면 바로 비디오 플레이어가 보인다.
2. 업로드되는 비디오 포맷은 MP4(H.264)이다.
3. 기존 WebM 녹화·저장·Bugshot 내 미리보기 로직은 변경하지 않는다. 변환은 Jira 제출 시점에만 수행한다.

## 비목표 (Non-goals)

- GitHub/Linear/Notion 제출 흐름 변경 (각 플랫폼은 기존 WebM 첨부 유지)
- 녹화 포맷 자체를 MP4로 변경 (MediaRecorder는 WebM 유지)
- Bugshot 사이드 패널의 비디오 미리보기 UI 변경
- 오디오 트랙 처리 (현재 `audio: false`로 녹화)

## 사용자 시나리오

### 메인 플로우
1. 사용자가 화면을 녹화하고 이슈 초안을 작성한다 (기존과 동일).
2. Jira로 이슈를 제출한다.
3. 제출 과정에서 WebM → MP4 변환이 자동으로 수행된다.
4. Jira 이슈가 생성되고, 본문 "미디어" 섹션에 비디오가 인라인 플레이어로 보인다.

### 엣지 케이스
- **변환 실패**: WebCodecs API 미지원 또는 인코딩 오류 시, WebM을 그대로 첨부파일로 업로드하고 ADF에는 기존 텍스트 방식 유지 (graceful fallback).
- **긴 영상 (최대 60초)**: 변환에 수 초~십수 초 소요. 제출 버튼의 기존 로딩 상태가 커버.
- **DraftDetailDialog 재제출**: 이슈 목록에서 미제출 드래프트를 Jira로 제출할 때도 동일하게 변환.

## 성공 기준

- Jira 이슈 본문에서 비디오가 인라인 플레이어로 렌더링된다.
- 변환 실패 시에도 이슈 생성이 정상 완료된다 (비디오는 첨부파일로 존재).
- 기존 스크린샷/element 캡처의 Jira 제출에 회귀가 없다.
- GitHub/Linear/Notion 제출에 영향이 없다.
