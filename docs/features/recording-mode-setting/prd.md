# 녹화 모드 설정

## 배경

화면 녹화(`getDisplayMedia`)가 추가되면서 캡처 진입 화면의 버튼 그리드가 `1×2×2×1`(4행)로 늘어났다. Row 3에 [탭 녹화][화면 녹화] 두 버튼이 나란히 놓이고, 30초 리플레이가 단독 4행으로 밀려났다.

이 변경은 **명시적 트레이드오프**다 — 한 녹화 모드를 설정 뒤로 숨기는 대신 그리드를 `1×2×2`(3행)로 되돌리고 리플레이를 같은 행에 올린다. 동시 노출의 비용(4행 + 매 진입마다 모드 선택)을, "사용자가 보통 한 모드를 반복 사용하고 모드 전환은 드물다"는 가정 위에서 숨김의 비용(전환 시 설정 왕복)보다 크다고 본다. 이 가정은 데이터로 검증된 것이 아니라 제품 판단이며, 모드 전환이 잦은 사용자에게는 마찰이 늘어난다(설정 → Tabs → 캡처 화면 복귀). split 버튼의 ⚙로 설정 진입을 1클릭으로 줄여 이 마찰을 완화한다.

## 목표

- 설정 > 이슈 설정 > "캡처 설정" 섹션에 **녹화 모드 선택**(탭 녹화 / 화면 녹화) 컨트롤을 추가한다. shadcn **Tabs**(2개 탭을 값 컨트롤로). 선택값은 영속화된다.
- 캡처 진입 화면 그리드를 `1×2×2`로 되돌린다. Row 3 = [선택된 녹화 모드][30초 리플레이].
- Row 3의 녹화 버튼은 선택된 모드의 **기존 아이콘·i18n 라벨을 그대로** 사용한다 (탭=`AppWindow`/`issue.mode.video`, 화면=`MonitorPlay`/`issue.mode.screenRecord`).
- 녹화 버튼 **우측 안쪽에 ⚙ 아이콘 버튼**(단일 아이콘, 라벨 없음)을 absolute로 겹쳐(`[ 탭 녹화        ⚙ ]`), 캡처 화면에서 모드를 바꾸러 설정으로 1클릭에 갈 수 있게 한다. 녹화 버튼 본체와 ⚙는 클릭 영역이 분리된다(⚙ 클릭은 녹화를 시작하지 않음).

## 비목표 (Non-goals)

- 녹화 모드를 캡처 화면에서 직접 토글하는 인라인 컨트롤(드롭다운·세그먼트)은 만들지 않는다 (설정으로만 변경, 그리고 split 버튼의 ⚙로 이동). → 대안 검토(design) 참조.
- 화면 녹화 / 탭 녹화 각각의 캡처 로직(`video-capture.ts`, `video-recorder.ts`)은 변경하지 않는다 — 그리드 버튼이 어느 함수를 호출할지 분기만 추가한다.
- **캡처 단축키는 이번 범위 밖이다.** `capture-video`(Cmd/Ctrl+Shift+X)는 **기존대로 탭 녹화 고정**을 유지한다(분기 추가 안 함). 단축키→화면 녹화는 `chrome.commands`→sidepanel 경로에서 transient user activation이 전파되지 않아 `getDisplayMedia` picker가 뜨지 않으므로 의도적으로 다루지 않는다. 캡처 단축키 3개 제거는 **별도 feature**로 진행한다.
- 녹화 모드는 **전역 단일값**이다 — 플랫폼·탭별로 따로 기억하지 않는다.
- 녹화 산출물 포맷·해상도·프레임레이트 설정은 이번 범위 밖이다.
- 권한 변경 없음 — 탭 녹화(`tabCapture`)·화면 녹화(`getDisplayMedia`, web 표준) 모두 기존 권한 범위.

## 사용자 시나리오

### 시나리오 1: 녹화 모드 변경
1. 설정 탭 → 이슈 설정 → "캡처 설정" 섹션
2. "녹화 모드" 행의 Tabs에서 [탭 녹화 | 화면 녹화] 중 하나 선택
3. 선택 즉시 영속화되고, 캡처 진입 화면 Row 3 녹화 버튼이 해당 모드의 아이콘·라벨로 바뀐다

### 시나리오 2: 캡처 화면에서 녹화 시작
1. 디버그 → 이슈 작성 진입 화면
2. Row 3 녹화 버튼(선택된 모드) 클릭 → 해당 모드 녹화 시작

### 시나리오 3: 캡처 화면에서 모드 전환
1. 캡처 진입 화면에서 녹화 버튼 옆 ⚙ 아이콘 클릭
2. 설정 탭(이슈 설정 sub-tab)이 열린다 — 캡처 설정 섹션은 sub-tab 상단이라 별도 스크롤 없이 보인다
3. 모드 변경 후 사용자가 디버그 탭으로 돌아오면 녹화 버튼이 전환돼 있다 (설정→캡처 자동 복귀는 없음, 편도 이동)

### 엣지 케이스
- **기존 사용자**: `recordingMode` 설정값이 없으므로 마이그레이션으로 기본값 `tab`을 부여 → 기존 동작(탭 녹화)과 동일하게 보인다.
- **녹화 진행 중 모드 변경**: `recordingMode`(영속 설정)와 진행 중 녹화 세션(`editor-store.recordingSource`)은 직교다. 녹화 중 설정에서 모드를 바꿔도 진행 중 녹화에는 영향 없고 **다음 녹화부터** 반영된다. 단, 캡처 진입 화면(`EmptyState`)은 녹화 중에는 렌더되지 않으므로(녹화 상태 화면으로 전환) 실사용 충돌 가능성은 낮다.

## 성공 기준

- 캡처 진입 화면이 정확히 3행(`1×2×2`)으로 렌더링되고, Row 3에 [녹화 버튼(우측 안 ⚙ 오버레이)][리플레이]가 같은 행에 놓인다. 좁은 폭(실 가용 ~288px)에서 라벨이 잘려도(`truncate`, ⚙ 자리만큼 우측 패딩) 아이콘과 ⚙ 클릭 영역은 유지된다.
- 설정에서 모드를 바꾸면 캡처 화면 녹화 버튼의 아이콘·라벨·동작이 즉시 전환된다 (store 라이브 구독, 리로드 불필요).
- 녹화 버튼 클릭이 선택된 모드를 시작한다. (단축키는 이번 범위 밖 — 기존 탭 녹화 고정 유지.)
- ⚙ 버튼이 설정 탭(이슈 설정 sub-tab)을 연다.
- 기존 사용자의 첫 로드 시 탭 녹화로 동작한다 (마이그레이션).
- ko/en i18n 키 대칭, `pnpm test` / e2e green.
