# 녹화 모드 설정

## 배경

화면 녹화(`getDisplayMedia`)가 추가되면서 캡처 진입 화면의 버튼 그리드가 `1×2×2×1`(4행)로 늘어났다. Row 3에 [탭 녹화][화면 녹화] 두 버튼이 나란히 놓이고, 30초 리플레이가 단독 4행으로 밀려났다. 버튼이 많아 시각적으로 무겁고, 두 녹화 모드를 매번 화면에서 고르는 것은 대부분의 사용자가 한 가지 모드만 반복 사용하는 실제 패턴과 맞지 않는다.

녹화 모드를 **설정에서 한 번 정해두고**, 캡처 화면에는 선택된 모드 하나만 노출하면 그리드를 `1×2×2`(3행)로 되돌리면서 리플레이를 같은 행에 둘 수 있다.

## 목표

- 설정 > 이슈 설정 > "캡처 설정" 섹션에 **녹화 모드 선택**(탭 녹화 / 화면 녹화) 컨트롤을 추가한다. 선택값은 영속화된다.
- 캡처 진입 화면 그리드를 `1×2×2`로 되돌린다. Row 3 = [선택된 녹화 모드][30초 리플레이].
- Row 3의 녹화 버튼은 선택된 모드의 **기존 아이콘·i18n 라벨을 그대로** 사용한다 (탭=`AppWindow`/`issue.mode.video`, 화면=`MonitorPlay`/`issue.mode.screenRecord`).
- 녹화 버튼에 **설정으로 이동하는 아이콘 버튼**(단일 아이콘, 라벨 없음)을 붙여, 캡처 화면에서 모드를 바꾸러 설정으로 한 번에 갈 수 있게 한다.
- 캡처 단축키 `capture-video`(Cmd/Ctrl+Shift+X)는 **선택된 녹화 모드**를 시작한다.

## 비목표 (Non-goals)

- 녹화 모드를 캡처 화면에서 직접 토글하는 인라인 컨트롤은 만들지 않는다 (설정으로만 변경, 그리고 split 버튼의 ⚙로 이동).
- 화면 녹화 / 탭 녹화 각각의 캡처 로직(`video-capture.ts`, `video-recorder.ts`)은 변경하지 않는다 — 어느 함수를 호출할지 분기만 추가한다.
- 모드별 독립 단축키는 추가하지 않는다 (단축키 4개 = Chrome 상한, CLAUDE.md 게이트웨이).
- 권한 변경 없음 — 탭 녹화(`tabCapture`)·화면 녹화(`getDisplayMedia`, web 표준) 모두 기존 권한 범위.

## 사용자 시나리오

### 시나리오 1: 녹화 모드 변경
1. 설정 탭 → 이슈 설정 → "캡처 설정" 섹션
2. "녹화 모드" 행에서 [탭 녹화 | 화면 녹화] 세그먼트 중 하나 선택
3. 선택 즉시 영속화되고, 캡처 진입 화면 Row 3 녹화 버튼이 해당 모드의 아이콘·라벨로 바뀐다

### 시나리오 2: 캡처 화면에서 녹화 시작
1. 디버그 → 이슈 작성 진입 화면
2. Row 3 녹화 버튼(선택된 모드) 클릭 → 해당 모드 녹화 시작
3. 또는 단축키 Cmd/Ctrl+Shift+X → 선택된 모드 녹화 시작

### 시나리오 3: 캡처 화면에서 모드 전환
1. 캡처 진입 화면에서 녹화 버튼 옆 ⚙ 아이콘 클릭
2. 설정 > 이슈 설정 캡처 섹션으로 이동
3. 모드 변경 후 다시 캡처 화면으로 돌아오면 녹화 버튼이 전환돼 있다

### 엣지 케이스
- **기존 사용자**: `recordingMode` 설정값이 없으므로 마이그레이션으로 기본값 `tab`을 부여 → 기존 동작(탭 녹화)과 동일하게 보인다.
- **화면 녹화 + 단축키**: `getDisplayMedia`는 user activation을 요구한다. 단축키 경로에서 activation이 전파되는지 구현 시 실측한다 (위험 요소, design 참조).

## 성공 기준

- 캡처 진입 화면이 정확히 3행(`1×2×2`)으로 렌더링되고, Row 3에 [녹화 버튼(+⚙)][리플레이]가 같은 행에 놓인다.
- 설정에서 모드를 바꾸면 캡처 화면 녹화 버튼의 아이콘·라벨·동작이 즉시 전환된다.
- 녹화 버튼 클릭과 단축키 모두 선택된 모드를 시작한다.
- ⚙ 버튼이 설정 캡처 섹션으로 이동시킨다.
- 기존 사용자의 첫 로드 시 탭 녹화로 동작한다 (마이그레이션).
- ko/en i18n 키 대칭, `pnpm test` / e2e green.
