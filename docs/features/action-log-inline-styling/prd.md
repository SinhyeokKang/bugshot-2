# 액션 로그 인라인 텍스트 디자인 강화

## 배경
액션 로그(`ActionLogContent`)의 각 항목은 현재 `"값" 따옴표`로만 변수 부분을 구분한 평문 한 줄이다. 입력한 값, 클릭한 요소, 키 조합이 주변 텍스트와 시각적으로 섞여 빠르게 훑기 어렵다. 버그 재현 흐름을 검토할 때 "무엇을 입력/클릭/이동했는가"가 한눈에 들어와야 한다.

레퍼런스(`/Users/sinhyeokkang/Desktop/스크린샷 2026-06-25 15.06.25.png`)는 변수 부분을 박스 칩·문법 하이라이팅·링크로 구분해 가독성을 높인 형태를 보여준다.

## 목표
- 사용자가 **입력·선택한 값**(input 텍스트, select 옵션, 키 조합)을 monospace **박스 칩**으로 강조한다.
- 클릭 대상에 **접근성 이름이 없을 때** 요소를 `<tag type="...">` 형태로 **문법 하이라이팅**한다(태그명·속성·값·괄호 색 분리).
- 클릭 대상에 **접근성 이름이 있으면** 그 이름을 강조 텍스트로 보여준다(칩 아님).
- 네비게이션 URL은 기존 blue+underline 링크를 유지한다.
- ko/en 동사 어순을 그대로 유지한다(템플릿 문자열 불변).
- 사이드패널과 로그 뷰어(`dist-log-viewer`) 양쪽에 동일하게 적용된다(공유 컴포넌트).

## 비목표 (Non-goals)
- 콘솔/네트워크 로그 항목의 렌더링 변경(액션 로그 한정).
- 클릭 대상에 `type` 외 다른 속성(`name`/`role`/`id`/`href` 등) 노출.
- 클릭 대상에 접근성 이름과 태그를 **동시** 표시.
- 액션 로그 JSON/HTML export 포맷 변경.
- 새 i18n 키 추가 또는 기존 동사 템플릿 문자열 변경.
- 마스킹 정책·캡처 대상·필터·검색 동작 변경.

## 사용자 시나리오
액션 녹화 후 사이드패널 또는 로그 뷰어의 "액션" 탭을 연다.

1. **텍스트 입력** → `Entered [cheese] in "Email"` — 값 `cheese`가 박스 칩.
2. **비밀번호 입력** → `Entered [⌷********⌷] in "Password"` — 마스킹 값은 점선 테두리 muted 칩.
3. **select 선택** → `Selected [Option 1] in "Category"` — 옵션 텍스트가 칩.
4. **단축키** → `Pressed [Ctrl+S]` — 키 조합이 칩.
5. **이름 있는 클릭**(버튼 텍스트 "Save") → `Clicked Save` — 강조 텍스트.
6. **이름 없는 클릭**(빈 div, 아이콘 버튼 등) → `Clicked <div>` 또는 `<input type="text">` — 문법 하이라이팅 태그.
7. **페이지 이동** → `Navigated to https://...` — URL이 blue+underline 링크(새 탭).
8. **토글**(체크박스/라디오) → `Checked "Remember me"` — 필드 라벨은 기존 따옴표 유지(칩 아님).

엣지 케이스:
- 좁은 사이드패널 폭에서 칩·긴 값이 줄바꿈돼도 레이아웃이 깨지지 않는다.
- 마이그레이션 전 저장된 세션의 클릭 항목(태그 정보 없음)은 기존처럼 이름(없으면 selector)을 따옴표로 표시한다(회귀 없음).

## 성공 기준
- 위 8개 시나리오가 의도대로 렌더된다(수동 + e2e).
- `pnpm typecheck`·`pnpm test` 통과, i18n locales 대칭 테스트 통과.
- 로그 뷰어 빌드(`pnpm build:log-viewer`) 산출물에서도 동일 렌더.
- 기존 저장 세션 로드 시 클릭 항목이 깨지지 않는다.
