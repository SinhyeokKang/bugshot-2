# Replay Trim 탭 리팩터

## 배경
현재 30s Replay 트림 오버레이(`ReplayTrimDialog`)는 가운데 영상 영역(`<video>`) + 하단 트림 타임라인 구조다. 콘솔·네트워크·동작 로그는 상단 ButtonGroup의 아이콘 버튼을 눌러 **모달 다이얼로그**로만 확인할 수 있다. 그래서:
- 트리밍하면서 어떤 로그가 잘려나갈지 한눈에 볼 수 없다. 다이얼로그를 열면 타임라인이 가려져 트림 핸들을 동시에 조작할 수 없다.
- 로그를 보려면 매번 다이얼로그를 열고 닫아야 해 영상↔로그 전환 비용이 크다.

## 목표
- 트림 오버레이에 **아이콘 4단 탭**([영상] [콘솔] [네트워크] [동작])을 추가한다. 탭은 기존 로그 선택 ButtonGroup을 대체한다.
- 영상 탭은 현재와 동일(가운데 영상 영역에서 재생).
- 콘솔/네트워크/동작 탭은 영상 영역을 **로그 뷰어와 동일한 로그 UI**(`*LogContent`)로 대체한다. 로그 뷰어처럼 각 로그에 상대 timestamp가 표시된다(콘솔·네트워크·동작 모두). 이는 기존 `syncBaseMs` prop(=videoStartedAt) 재사용으로 켜지며 LogContent 동작 변경이 아니다.
- 트리밍 타임라인·재생 버튼·언두·리두·취소·제출은 **모든 탭에서 전역으로** 보이고 동작한다(콘솔 탭에서도 트리밍 가능).
- 로그 탭에서 **잘려나갈 후보 로그를 muted(흐림) 처리**해, 현재 트림 구간 기준으로 무엇이 제거될지 실시간으로 보여준다.
  - 각 로그 row가 트림 핸들 조작에 따라 실시간 갱신된다.
  - muted 흐림은 로그의 레벨/상태 색상과 병렬로 적용돼도 둘 다 명확히 구분된다.
- 타임라인 에러 마커 클릭 시 다이얼로그 대신 해당 로그 탭으로 전환 + 그 로그로 스크롤한다.

## 비목표 (Non-goals)
- 로그 탭에서 재생 위치(playhead)에 맞춰 현재 로그 행을 하이라이트/자동스크롤하는 동기화는 하지 않는다(정적 리스트 + muted만).
- `ConsoleLogContent`/`NetworkLogContent`/`ActionLogContent`의 기존 필터·검색·origin 필터·상세 패널 동작은 변경하지 않는다(그대로 재사용). 상대 timestamp 표시는 기존 `syncBaseMs` prop으로 켜는 것이라 동작 변경이 아니며, 신규 `isMuted` optional prop만 추가한다.
- 실제 트림 적용 로직(`applyReplayTrim`)의 동작·결과는 변경하지 않는다(경계 계산 헬퍼만 추출, 결과 동일).
- 동작(action) 로그용 타임라인 마커는 추가하지 않는다(현재대로 console/network만).
- `ConsoleLogPreviewDialog`/`NetworkLogPreviewDialog`/`ActionLogPreviewDialog` 컴포넌트 자체는 삭제하지 않는다(DraftDetailDialog·PreviewPanel·DraftingPanel에서 계속 사용).

## 사용자 시나리오
1. 30s Replay 캡처 → 트림 오버레이 진입. 기본 **영상 탭**, 가운데 영상이 보이고 하단에 트림 타임라인·재생·언두/리두·취소/제출이 있다.
2. 상단 탭에서 **콘솔** 아이콘을 누른다 → 영상 영역이 콘솔 로그 리스트로 바뀐다. 하단 트림 타임라인·액션바는 그대로 남아 있다.
3. 트림 핸들을 안쪽으로 드래그한다 → 선택 구간 밖에 떨어지는 콘솔 로그 row들이 **즉시 흐려진다(muted)**. 핸들을 움직일 때마다 흐림 범위가 실시간으로 갱신된다.
4. **네트워크** 탭으로 전환 → 동일한 muted 미리보기. 각 요청에 `0:03` 같은 상대 시각이 붙어 있다.
5. 타임라인 위 에러 마커를 클릭 → 해당 로그 탭으로 자동 전환되고 그 로그로 스크롤된다.
6. **제출**(체크)을 누르면 현재 구간으로 영상·로그가 잘린다. 흐림으로 보였던 로그가 실제로 제거된다.

### 엣지 케이스
- 전체 구간 선택(트림 안 함): muted 로그 0개. 제출해도 재인코딩·로그 trim 없음(`isFullRange` early-return과 일치).
- 로그가 없는 탭: 해당 탭은 비활성(disabled) — 현재 ButtonGroup 버튼의 `disabled={!log}`와 동일.
- 가드밴드(`REPLAY_LOG_GUARD_MS`): 시작을 안 자른 경우 첫 프레임 1.5s 이전 로그도 보존된다. muted 미리보기는 실제 trim과 동일한 경계를 써 이 보존을 그대로 반영한다.

## 성공 기준
- 4개 탭이 아이콘으로 렌더되고 전환된다. 영상 탭은 기존과 픽셀 동일.
- 로그 탭에서 트림 핸들 드래그 시 잘려나갈 row가 흐려지고, 제출 후 실제 제거되는 로그 집합과 **정확히 일치**한다.
- 트림/재생/언두/리두/취소/제출이 모든 탭에서 동작한다.
- 마커 클릭이 탭 전환 + 스크롤로 동작한다.
- muted 흐림과 레벨/상태 색상이 동시에 적용돼도 시각적으로 구분된다.
- `pnpm test`(경계 헬퍼 단위 테스트 포함)·`pnpm typecheck` 통과.
