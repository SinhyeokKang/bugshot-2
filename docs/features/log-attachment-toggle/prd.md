# 로그 첨부 단일 토글 (Log Attachment Single Toggle)

## 배경

이슈 드래프트 단계에는 캡처된 로그(network/console/action)를 이슈에 첨부할지 정하는 UI가 있다. 현재는 **로그 타입마다 카드 1개 + 스위치 1개**(총 3개)로 개별 on/off를 받는다.

이 3개 토글은 **과거 잔재**다. 예전엔 로그를 타입별 HAR/JSON 파일로 각각 첨부했기에 타입별 토글이 의미가 있었다. 지금은:

- 이슈에 붙는 로그 산출물은 `buildCaptureFiles` → `buildLogsHtml`가 만드는 **단일 `logs.html` 하나**다(`src/sidepanel/lib/buildCaptureFiles.ts`).
- HAR/JSON 변환은 제출 경로가 아니라 **log-viewer(logs.html) 안의 다운로드 버튼**에서만 일어난다(`src/log-viewer/App.tsx`).

즉 3개 토글은 "한 파일 안에 어떤 타입을 넣을지"만 조절하는데, 실사용에서 타입별로 쪼개 넣을 이유가 거의 없고 UI만 복잡하다. 상세 다이얼로그도 타입별로 3개(`NetworkLogPreviewDialog`/`ConsoleLogPreviewDialog`/`ActionLogPreviewDialog`)로 갈려 있어 중복이 크다.

## 목표

1. 로그 타입별 3개 첨부 토글을 **단일 토글 하나**로 합친다 — logs.html 첨부를 통째로 on/off.
2. 첨부 카드를 **단일 카드 하나**로 합치고, 카드 설명(desc)에 **타입별 캡처 건수(+에러 건수)**를 보여준다.
3. 타입별 3개 상세 다이얼로그를 **탭형 단일 다이얼로그** 하나로 합친다(console/network/action 탭). 푸터 액션은 **첨부/첨부 해제**.
4. 위 변경이 제출 산출물(logs.html)과 저장된 이슈 상세 렌더를 깨뜨리지 않는다.

## 비목표 (Non-goals)

- 타입별 개별 첨부 기능 보존 — 명시적으로 폐기한다(all-or-nothing으로 단순화).
- 첨부 토글을 설정 탭의 영구 설정으로 승격 — **세션별 상태 유지**(리포트마다 on/off, drafting 진입 시 기본 on). 현재 UX와 동일.
- 본문 인라인 로그 삽입 다이얼로그(`LogInsertDialog`) 기능 변경 — 그대로 둔다(별개 기능: 로그 1건을 본문 코드블록으로 삽입).
- log-viewer(logs.html) 내부의 HAR/JSON 다운로드 버튼 변경.
- `buildLogsHtml`/`buildCaptureFiles`의 로그 emit 로직 변경.
- 저장된 이슈 레코드의 per-type blob 키(`networkLogBlobKey`/`consoleLogBlobKey`/`actionLogBlobKey`) 스키마 변경 — 저장 계층은 그대로, UI 토글만 단일화.

## 사용자 시나리오

### 시나리오 1: 스크린샷/영상/자유형 캡처 후 드래프트

1. 사용자가 스크린샷(또는 영상·자유형)을 캡처하고 drafting 화면에 진입한다.
2. 로그 섹션에 **카드 1개**("로그")가 뜬다. 스위치는 기본 **on**.
3. 카드 desc에 "네트워크 12(에러 3) · 콘솔 5(에러 2) · 액션 8"처럼 캡처된 타입별 건수가 보인다. 캡처 안 된 타입은 표기에서 빠진다.
4. 사용자가 카드 본문을 클릭하면 **탭형 로그 다이얼로그**가 열린다. 상단 탭으로 console/network/action을 오간다(캡처된 타입 탭만 노출).
5. 다이얼로그 푸터에서 **[첨부 해제]**를 누르면 스위치가 off로 바뀌고 닫힌다(또는 카드 스위치로 직접 off).
6. off면 제출 시 logs.html이 첨부되지 않는다. on이면 캡처된 모든 타입이 담긴 logs.html 하나가 첨부된다.

### 시나리오 2: 미리보기(Preview) — 읽기 전용

1. 드래프트 확정 후 미리보기 화면(`PreviewPanel`)에서 첨부 예정 로그를 확인한다.
2. 첨부가 on이고 캡처 로그가 있으면 **단일 카드**(읽기 전용, 스위치 없음)가 보인다.
3. 카드 클릭 시 탭형 다이얼로그가 읽기 전용(첨부/해제 버튼 없음, 닫기만)으로 열린다.

### 시나리오 3: 저장된 이슈 상세(DraftDetailDialog) — 읽기 전용

1. 이슈 목록에서 저장/제출된 이슈의 상세를 연다.
2. 실제 첨부된(=blob 키가 있는) 로그 타입들이 **단일 카드**로 요약되고, 클릭하면 탭형 다이얼로그(읽기 전용)로 상세를 본다.
3. 과거 데이터(일부 타입만 첨부된 레코드)도 존재하는 타입만 탭/건수에 반영해 정상 렌더된다.

### 엣지 케이스

- 캡처 로그가 하나도 없으면(모든 타입 captured 0) 카드·다이얼로그를 렌더하지 않는다(현행과 동일).
- element 캡처 모드는 로그를 수집하지 않으므로 로그 카드가 없다(현행과 동일).
- 세션 복원(패널 닫았다 다시 열기): 첨부 토글 상태와 카드 건수가 유지된다. (현행은 off 후 재오픈 시 해당 타입 카드가 사라지는 버그가 있었는데, 단일화하며 로그 데이터를 첨부 상태와 무관하게 복원해 개선한다.)

## 성공 기준

- 드래프트 로그 섹션에 카드가 **정확히 1개** 렌더되고, 스위치 1개로 logs.html 첨부 전체가 on/off 된다.
- 카드 desc가 캡처된 타입별 건수와 에러 건수를 정확히 보여준다.
- 카드 클릭 시 열리는 다이얼로그가 캡처된 타입 탭만 노출하고, 탭 전환으로 각 타입 상세를 본다.
- 첨부 on/off에 따라 제출 시 logs.html 첨부 여부가 정확히 갈린다(기존 `selectAttachedLogs`/`buildEditorLogsCaptureInput` 동작과 결과 동일 — 단, 타입별이 아니라 통짜로).
- Preview/DraftDetailDialog 읽기 전용 경로가 단일 카드+탭 다이얼로그로 정상 동작한다.
- `pnpm test` 통과, `pnpm typecheck` 통과.
