# Log Viewer — Screenshot 좌측 패널

## 배경

이슈에 첨부되는 `logs.html`(로그 뷰어)은 video 모드일 때 좌측에 영상 플레이어를, 우측에 로그 탭을 나란히 보여준다. 반면 screenshot 모드에서는 `LogViewerData.video`가 `null`이라 좌측 패널 없이 로그만 전폭으로 렌더된다(`src/log-viewer/App.tsx:203-205`). 캡처한 스크린샷은 이슈 본문 첨부 이미지로만 존재해, 로그를 보는 화면에서 "어떤 화면을 캡처했는지" 맥락을 함께 볼 수 없다.

## 목표

- screenshot 모드로 첨부한 `logs.html`을 열면 좌측 패널에 캡처한 스크린샷이 표시된다.
- 좌측 패널의 시각 스타일(리사이즈 분할, 검정 배경, `object-contain` 중앙 정렬, 상단 이슈 타이틀 오버레이)은 video 플레이어와 동일하다.
- 단, 하단 영상 컨트롤 영역(타임라인 ProgressBar, 재생/일시정지, 시간 표시, 다운로드 버튼)은 표시하지 않는다.
- 스크린샷은 단일 파일이므로 캐러셀·다중 이미지 네비게이션은 만들지 않는다.

## 비목표 (Non-goals)

- element 모드(before/after 2장)·freeform 모드에 좌측 미디어 패널을 추가하지 않는다. 이번 스코프는 **screenshot 모드 전용**.
- 로그가 하나도 캡처되지 않아 `logs.html`이 생성되지 않는 경우의 동작은 바꾸지 않는다(아래 시나리오 참조). 스크린샷만 보여주려고 `logs.html`을 새로 생성하지 않는다.
- 스크린샷 줌·팬·라이트박스·다운로드 같은 인터랙션을 추가하지 않는다(정적 표시).
- 로그 마커/타임라인 동기화(seek)는 시간축이 없는 정적 이미지엔 적용하지 않는다.

## 사용자 시나리오

1. 사용자가 screenshot 모드로 영역을 캡처하고(콘솔·네트워크 로그도 함께 수집됨) 이슈를 등록한다.
2. 이슈에 `screenshot.webp`(본문 이미지)와 `logs.html`이 첨부된다.
3. 다른 사람이 이슈의 `logs.html`을 브라우저로 연다.
4. 좌측 60%에 캡처한 스크린샷이 검정 배경 위 `object-contain`으로 표시되고, 우측 40%에 콘솔/네트워크 로그 탭이 나타난다.
5. 좌측 패널에 마우스를 올리면 상단에 이슈 타이틀·키 오버레이가 페이드 인 된다(video와 동일).
6. 좌측 패널 하단에는 어떤 컨트롤 바도 없다.

### 엣지 케이스

- **로그가 비어 `logs.html`이 생성되지 않음**: 기존과 동일하게 좌측 패널 없이 아무 `logs.html`도 없음. 스크린샷은 본문 첨부 이미지로만 존재. (게이팅 유지)
- **스크린샷 이미지 로드 실패**: 좌측 패널에 에러 문구를 표시(video의 `logViewer.video.error`와 대칭).
- **annotated 스크린샷**: 본문 첨부와 동일하게 `screenshotAnnotated ?? screenshotRaw` 우선순위 이미지를 표시(주석이 그려진 버전).

## 성공 기준

- screenshot 모드 + 로그 존재 시 생성된 `logs.html`을 열면 좌측에 스크린샷, 우측에 로그가 60/40 리사이즈 분할로 보인다.
- 좌측 패널에 하단 컨트롤 바가 없다.
- video 모드 동작은 기존 그대로(회귀 없음).
- 로그가 없는 screenshot 모드는 기존 그대로 `logs.html` 미생성.
- `pnpm test`의 `buildCaptureFiles` 단위 테스트가 screenshot 임베드/video 비임베드를 검증하며 통과.
