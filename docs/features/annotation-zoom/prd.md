# 어노테이션 캔버스 줌

## 배경

v1.5.6에서 페이지 전체 캡처(scroll capture)가 들어오면서, 어노테이션 오버레이가 감당해야 하는 이미지의 종횡비가 크게 달라졌다.

현재 `AnnotationOverlay`는 이미지를 **폭·높이 모두 가용 영역에 맞추는 배율**(`fitScale`, 최대 1 — 확대 불가) 하나로만 표시한다. 캡처 파이프라인은 `MAX_OUTPUT_PIXELS = 4_000_000` 캡을 걸어 최종 이미지를 다운스케일하므로, 세로로 긴 페이지 전체 캡처는 예컨대 1074×3730 같은 이미지가 된다. 사이드패널 폭 400px 기준(캔버스 뷰포트에 가로 패딩은 없고, 세로 스크롤바가 뜨면 ~390px)에서 이 이미지의 표시 배율은:

- 현행 fit-all: `min(availW/1074, 0.7×availH/3730, 1)` ≈ **15%**
- fit-width: `min(availW/1074, 1)` ≈ **36%**

15%에서는 본문 텍스트도, 버그가 난 UI 요소도 식별되지 않는다. 어노테이션(화살표·박스·텍스트)을 **정확한 위치에 찍는 것 자체가 불가능**하다. 확대 수단이 없으면 페이지 전체 캡처는 사실상 어노테이션할 수 없는 이미지다.

> 이 문서의 배율 수치(15%·36% 등)는 400px 패널 + 1074×3730 이미지를 가정한 **예시**다. 실제 값은 뷰포트 실측 폭(`ResizeObserver`)에서 파생된다. 현행 코드의 `window.innerWidth - 32`·`window.innerHeight * 0.7`은 컨테이너 실측이 아닌 눈대중 매직넘버이며 이번 변경으로 사라진다.

## 목표

- 어노테이션 오버레이의 캔버스를 사용자가 확대·축소할 수 있다. 배율 범위는 전체(fit-all) ~ 400%.
- **화면 밖으로 넘치는 이미지를 선택 도구로 드래그해 이동(팬)할 수 있다.** 마우스가 없어도 캔버스 뷰포트에 키보드 포커스를 줘 화살표키로 스크롤할 수 있다.
- 확대·축소는 **지금 보고 있는 지점(뷰포트 중앙)을 중심으로** 일어난다. 팬으로 중심을 옮겼으면 그 새 중심이 기준이 된다.
- 확대 상태에서도 그리기·선택·변형·텍스트 입력이 원본 좌표 기준으로 정확히 동작한다.
- 진입 시 기본 배율을 fit-width로 바꿔, 페이지 전체 캡처가 처음부터 크게 보이게 한다(1074×3730 기준 15% → 36%).
- **전체 조망 수단을 남긴다** — fit-width로 진입해도 콤보박스의 `전체` 항목으로 이미지 전체를 한눈에 볼 수 있다(위치 파악용 네비게이션).
- 배율을 조작한 상태임을 사용자가 인지하고 한 번에 되돌릴 수 있다.
- 확대·축소해도 **완료(Done) 결과물의 해상도는 변하지 않는다** — export는 항상 원본 natural 해상도.

## 비목표 (Non-goals)

- Ctrl/Cmd + 휠 줌, 핀치 줌, 줌 키보드 단축키(+/-/0). 이번엔 버튼·콤보박스만. (단, 뷰포트 포커스 후 **화살표키 스크롤**은 네이티브 동작으로 지원한다)
- 손바닥(Hand) 도구, 스페이스+드래그 팬. 팬은 **선택 도구에서 빈 캔버스 드래그** 하나로만 건다. 그리기 도구 사용 중에는 스크롤·트랙패드로 이동한다.
- 커서 위치 기준 확대(zoom-to-cursor). 앵커는 뷰포트 중앙 고정.
- 배율·팬 위치의 세션 영속화. 오버레이를 닫으면 사라진다.
- 녹화 중 그리기 footer(`IssueTab`의 `recording-pen` 경로)와 element before/after 비교 뷰. 줌 대상 아님.
- **그리기·도형 드래그 중 캔버스 가장자리 자동 스크롤**(edge auto-scroll). 도형은 현재 뷰포트 안에서만 그리고 이동한다. 화면 밖 대상은 먼저 팬으로 끌어온 뒤 그린다.

## 사용자 시나리오

### 기본 플로우 — 페이지 전체 캡처 어노테이션

1. 사용자가 페이지 전체 캡처로 긴 페이지를 찍고, 작성 화면의 미디어 섹션에서 **주석 추가**를 누른다.
2. 어노테이션 오버레이가 열린다. **선택 도구가 활성 상태로 진입**하고, 이미지는 **fit-width 배율**(예: 36%)로 표시되며 캔버스 우상단 줌 컨트롤 라벨에 `36%`가 보인다. 좌상단 맞춤 버튼은 **보이지 않는다**(맞춤 상태이므로).
3. 세로가 넘치므로 **캔버스 빈 곳을 드래그해** 아래쪽으로 끌어 내린다(커서가 `grab`으로 바뀌어 있다). 버그가 난 영역을 화면 중앙에 놓는다.
4. 영역이 작아서 잘 안 보인다. `[+]`를 눌러 50% → 75% → 100%로 확대한다. **팬으로 중앙에 놓은 그 지점을 중심으로** 확대된다.
5. 배율이 맞춤에서 벗어났으므로 **좌상단에 맞춤 버튼이 나타난다.**
6. 100%에서 화살표 도구로 버그 요소를 가리키고 텍스트를 단다. 도형은 원본 좌표에 정확히 찍힌다.
7. 다시 선택 도구로 돌아와 도형을 잡아 끌면 **도형이 이동**하고, 빈 곳을 잡아 끌면 **화면이 이동**한다.
8. 전체를 다시 보려면 좌상단 맞춤 버튼을 누른다 → 배율이 36%로 돌아가고 맞춤 버튼이 다시 사라진다.
9. **완료**를 누르면 원본 해상도(1074×3730) webp가 나온다. 배율·팬은 결과물에 영향을 주지 않는다.

### 콤보박스로 배율 점프

- 줌 컨트롤 가운데 `36% ▾`를 클릭하면 프리셋 목록이 열린다: `전체(15%)` · `맞춤(36%)` · `50%` · `75%` · `100%` · `150%` · `200%` · `300%` · `400%`.
- **맨 위 `전체`는 fit-all 배율** — 이미지가 폭·높이 모두 화면에 들어오는 배율이다. 본문을 읽기 위한 게 아니라 **"지금 보는 곳이 이미지의 어디쯤인지" 파악하는 네비게이션 용도**다. fit-all과 fit-width가 같은 이미지(가로가 지배하는 대부분의 캡처)에서는 이 항목이 나타나지 않는다.
- **fit-width보다 작고 fit-all보다 큰 프리셋은 목록에 나타나지 않는다** — 위 예에서 25%는 fit-width(36%)보다 작아 빠진다. 축소 방향의 선택지는 `전체` 하나로 족하다.
- `맞춤`을 고르면 좌상단 맞춤 버튼을 누른 것과 같다.

### 엣지 케이스

- **폭·높이 모두 가용 영역에 들어오는 작은 이미지(요소 캡처 등)**: fit-width도 fit-all도 1(=100%)이다. 프리셋 목록은 `맞춤(100%)` · `150%` · `200%` · `300%` · `400%`가 되고(`전체`는 맞춤과 같아 중복 제거), 진입 화면은 현행과 동일하다. 이미지가 화면에 다 들어오면 **팬은 비활성**이고 커서도 `default`다.
- **폭이 좁고 세로로 긴 캡처(허용 리스크)**: 300×2000 같은 요소·영역 캡처는 현행 fit-all에서 진입 배율이 ~28%였으나, fit-width로 바뀌면 **100%(폭이 가용 폭보다 작아 1로 클램프)로 열려 이미지의 대부분이 화면 밖**이 된다. 즉 fit-width 전환의 영향은 페이지 전체 캡처에만 국한되지 않는다. 이를 수용하되, 그래서 **`전체` 프리셋과 팬이 이 케이스의 안전망**이다(진입 직후에도 팬이 동작하고, `전체`로 한 번에 조망할 수 있다).
- **맞춤 상태에서도 팬이 필요하다**: fit-width는 폭만 맞추므로 페이지 전체 캡처는 **진입 시점부터 세로가 넘친다.** 즉 팬 활성 조건은 "확대했는가"가 아니라 **"스크롤 여지가 있는가"**다. 그래서 오버레이는 **선택 도구가 활성인 채로 진입**한다(현행은 도구 미선택 상태로 열려 포인터 핸들러가 즉시 반환된다).
- **빈 곳 클릭 vs 빈 곳 드래그**: 선택 도구로 빈 곳을 *클릭*하면 기존대로 선택이 해제된다. *드래그*(3px 초과 이동)하면 팬이고, 선택은 유지된다.
- **도형 위 드래그**: 선택 도구에서 도형을 잡아 끌면 팬이 아니라 **도형 이동**이다(현행 동작 유지).
- **드래그가 캔버스를 벗어남**: 팬·그리기 드래그 중 포인터가 캔버스 밖으로 나가도 **pointer capture로 드래그가 유지**되고, 어디서 버튼을 놓든 정상 종료된다. 확대 상태에서 이미지 가장자리로 끄는 건 가장 흔한 팬 동작이므로 필수다.
- **패널 폭 변경**: 사용자가 배율을 조작한 상태(맞춤 아님)에서 사이드패널 폭을 바꿔도 **배율은 유지된다.** 맞춤 상태라면 새 폭에 맞춰 자동 refit되고, 라벨 숫자만 바뀐다.
- **텍스트 입력 중 줌·스크롤**: 텍스트 박스 입력 중 줌 버튼을 누르거나 캔버스를 스크롤·팬하면 textarea가 커밋된다. 좌표가 어긋난 textarea가 남지 않는다. (`position: fixed` textarea는 배율·스크롤 위치를 기준으로 좌표를 역산하므로, 커밋하지 않으면 즉시 어긋난다. 현행은 캔버스에 스크롤이 거의 없어 드러나지 않던 문제가, fit-width + 팬 도입으로 상시화된다.)
- **배율 경계**: 목록 최저 배율(`전체`, 없으면 `맞춤`)에서 `[-]`, 400%에서 `[+]`는 비활성(`disabled`).
- **확대 후 좌우 이동**: 이미지가 가용 폭보다 넓어지면 좌우로도 팬·스크롤된다. 콘텐츠가 컨테이너보다 클 때 왼쪽·위쪽이 잘려 도달 불가능해지지 않아야 한다.
- **줌 컨트롤이 가리는 영역(허용 리스크)**: 캔버스 우상단의 플로팅 줌 컨트롤(~110×32px)과 좌상단 맞춤 버튼 아래 픽셀에는 도형을 찍을 수 없다. 팬이 가능한 이미지는 밀어서 피할 수 있으나, 팬이 불가한 작은 이미지(요소 캡처)에서는 피할 수 없다. 수용한다.

## 성공 기준

- [ ] 페이지 전체 캡처 이미지에서 100% 이상으로 확대해 본문 텍스트를 읽을 수 있고, 그 상태로 찍은 화살표·박스가 완료 후 원본 이미지의 같은 위치에 남는다.
- [ ] 진입 시 페이지 전체 캡처의 표시 배율이 fit-width다(1074×3730·400px 패널 기준 15% → 36%).
- [ ] 진입 직후(선택 도구 활성·맞춤 상태)에 빈 캔버스를 드래그해 이미지를 이동할 수 있고, 도형을 드래그하면 도형이 이동한다(둘이 섞이지 않는다).
- [ ] 팬·그리기 드래그가 캔버스 밖에서 끝나도 정상 종료된다(도형이 커밋되고, 팬이 조기 종료되지 않는다).
- [ ] 빈 곳 클릭으로 선택 해제하는 기존 동작이 팬 도입 후에도 유지된다.
- [ ] 확대·축소 시 뷰포트 중앙 지점이 유지되고, 팬으로 옮긴 중심이 그다음 확대의 기준이 된다.
- [ ] 세로로 긴 이미지에서 콤보박스 `전체` 항목으로 이미지 전체가 한 화면에 들어온다.
- [ ] 맞춤 버튼이 "배율을 조작한 상태"에서만 노출되고, 누르면 맞춤 배율로 정확히 복귀한다.
- [ ] 캔버스 뷰포트에 키보드 포커스를 주고 화살표키로 스크롤할 수 있다.
- [ ] 완료 결과물의 해상도가 배율·팬과 무관하게 원본 natural 크기다.
- [ ] 기존 어노테이션 e2e(`annotation-overlay.spec.ts`)가 그대로 통과한다.
