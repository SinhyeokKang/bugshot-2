# Multi-platform Issue Status Change

## 배경

GitHub 이슈에 대해서는 이슈 목록의 상태 배지를 클릭해 상태를 변경할 수 있는 팝오버가 이미 구현되어 있다 (`GithubStatusBadge`). Jira, Linear, Notion으로 제출한 이슈에 대해서도 동일한 UX를 제공하여, 사용자가 BugShot을 떠나지 않고도 이슈 상태를 업데이트할 수 있게 한다.

## 목표

- Jira, Linear, Notion 이슈의 상태를 이슈 목록 배지 팝오버에서 변경할 수 있다.
- 각 플랫폼의 상태 체계(Jira 트랜지션, Linear 워크플로 상태, Notion 데이터베이스 상태 속성)를 지원한다.
- 기존 GitHub 상태 변경과 일관된 UX를 유지한다.

## 비목표 (Non-goals)

- Jira 트랜지션에 필수 필드(해결 유형 등) 입력 UI 제공: 이번 스코프에서는 필수 필드 없이 실행 가능한 트랜지션만 성공하고, 필수 필드가 있으면 에러 토스트로 안내.
- 이슈 상태 외 다른 필드(담당자, 우선순위 등) 변경.
- 이슈 목록 밖(이슈 생성 모달 등)에서의 상태 변경.

## 사용자 시나리오

### 시나리오 1: Jira 이슈 상태 변경

1. 이슈 목록에서 Jira로 제출한 이슈의 상태 배지를 클릭한다.
2. 팝오버가 열리며 이슈에서 사용 가능한 트랜지션 목록을 로딩한다.
3. 로딩 완료 후 트랜지션 목록이 표시된다 (각 항목은 타겟 상태명 + 카테고리 색상).
4. 원하는 상태를 클릭하면 배지에 로딩 스피너가 표시된다.
5. 트랜지션 성공 시 배지가 새 상태로 업데이트된다.
6. 트랜지션 실패(필수 필드 등) 시 "Jira에서 직접 변경해 주세요" 토스트가 표시된다.

### 시나리오 2: Linear 이슈 상태 변경

1. Linear 이슈의 상태 배지를 클릭한다.
2. 팝오버가 열리며 해당 팀의 워크플로 상태 목록을 로딩한다.
3. 로딩 완료 후 상태 목록이 표시된다 (각 항목은 상태명 + type 기반 색상).
4. 원하는 상태를 클릭하면 배지에 스피너가 표시된다.
5. 성공 시 배지가 새 상태로 업데이트된다.

### 시나리오 3: Notion 페이지 상태 변경

1. Notion 이슈의 상태 배지를 클릭한다.
2. 팝오버가 열리며 해당 데이터베이스의 상태 옵션 목록을 로딩한다.
3. 로딩 완료 후 상태 옵션이 표시된다 (각 항목은 옵션명 + color→category 색상).
4. 원하는 옵션을 클릭하면 배지에 스피너가 표시된다.
5. 성공 시 배지가 새 상태로 업데이트된다.

### 엣지 케이스

- **인증 만료**: 401 응답 시 기존 `onOAuthExpired` 플로우가 동작 (토스트 + 재연결 안내).
- **Jira 트랜지션 목록 비어 있음**: 현재 상태에서 이동 가능한 트랜지션이 없으면 팝오버에 빈 상태 메시지 표시.
- **Notion 상태 속성 없음**: 기존 "상태 속성 없음" 배지가 그대로 표시되며, 팝오버 미지원.
- **Notion databaseId 없음**: 팝오버 미지원, 정적 배지만 표시.
- **네트워크 에러**: 옵션 로딩 실패 시 팝오버 내 에러 표시, 상태 변경 실패 시 토스트.

## 성공 기준

- [ ] Jira 이슈 상태 배지 클릭 → 트랜지션 팝오버 → 상태 변경 가능
- [ ] Linear 이슈 상태 배지 클릭 → 워크플로 상태 팝오버 → 상태 변경 가능
- [ ] Notion 이슈 상태 배지 클릭 → 상태 옵션 팝오버 → 상태 변경 가능
- [ ] 상태 변경 후 배지가 새 상태를 즉시 반영
- [ ] 에러 발생 시 적절한 토스트 메시지
- [ ] 기존 GitHub 상태 변경 동작에 회귀 없음
