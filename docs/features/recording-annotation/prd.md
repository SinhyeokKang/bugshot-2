# 녹화 중 어노테이션 (그리기)

## 배경

Bugshot는 탭·화면 녹화로 버그 재현 영상을 만든다. 하지만 녹화 중 "여기 이 버튼이요", "이 영역이 깨져요" 같은 시각적 강조 수단이 없다. 마우스 커서로 가리키는 것만으로는 영상만 보는 사람에게 의도가 전달되지 않는다. Loom·Screencastify의 펜(pen) 도구처럼, 녹화 화면 위에 손으로 그려 강조하고 잠시 뒤 자동으로 사라지는 가벼운 그리기 기능이 필요하다.

## 목표

- 탭 녹화·화면 녹화가 진행되는 동안, 페이지 위에 자유 곡선(freehand)을 그릴 수 있다.
- 그린 획(stroke)은 그려진 뒤 약 3초 후 자동으로 페이드아웃되어 사라진다 — 별도 지우기 조작 불필요.
- 그리기 오버레이는 녹화 스트림(tabCapture / getDisplayMedia)에 그대로 캡처되어 영상에 남는다.
- **사이드패널의 펜 토글 버튼**으로 그리기 모드를 켜고 끈다(페이지 위에는 별도 툴바/버튼 없음). 끄면 페이지를 정상적으로 조작할 수 있다.
- 새 권한·env·외부 의존성 없이 기존 콘텐츠 스크립트 패턴(shadow DOM 오버레이)만으로 구현한다.

## 비목표 (Non-goals)

- 페이지 위 플로팅 툴바/온-페이지 컨트롤 — **제외**. 켜고 끄기는 사이드패널 버튼으로만.
- 색상 선택, 도형(사각형·화살표·원), 텍스트 삽입, 지우개, 실행취소(undo) — **제외**. 자유 펜 1색(고정색)만.
- 그림의 영속 저장·이슈 첨부 — 제외. 어노테이션은 순수 시각 효과이며 어디에도 저장하지 않는다.
- 30초 Replay(`captureVisibleTab` 폴링) 중 그리기 — 이번 스코프 제외 (탭 녹화·화면 녹화 중에만).
- 페이드 시간(3초) 사용자 설정 — 제외. 고정값.
- iframe 내부에서의 그리기 — 제외. picker와 동일하게 top frame 한정.
- 사이드패널 내 정적 스크린샷 주석(`AnnotationOverlay.tsx`)과의 통합 — 별개 기능, 무관.

## 사용자 시나리오

### 주 플로우
1. 사용자가 사이드패널에서 탭 녹화(또는 화면 녹화)를 시작한다.
2. 녹화가 시작되면 사이드패널 녹화 컨트롤(정지 버튼 옆)에 펜 토글 버튼이 나타난다. 페이지에는 아무 UI도 뜨지 않는다(투명 그리기 레이어만 대기).
3. 사이드패널 펜 버튼을 클릭 → 그리기 모드 ON. 페이지 커서가 crosshair로 바뀌고 페이지 클릭이 막힌다(오버레이가 포인터를 가로챈다).
4. 페이지에서 마우스를 눌러 드래그하면 지나간 궤적을 따라 선이 그려진다. 손을 떼면 한 획이 완성된다.
5. 완성된 획은 약 3초 유지된 뒤 서서히 투명해지며 사라진다. 그 사이 다른 획을 계속 그릴 수 있다(각 획은 자기 타이머로 개별 페이드).
6. 사이드패널 펜 버튼을 다시 클릭 → 그리기 모드 OFF. 이미 그려진 획들은 남은 타이머대로 계속 페이드된다. 페이지는 다시 정상 조작 가능.
7. 녹화를 정지(또는 자동 60초 종료·화면공유 중지·취소)하면 펜 버튼과 페이지 오버레이가 즉시 제거된다.

### 엣지 케이스
- **녹화 중 페이지 이동(내비게이션)**: SPA·풀 내비게이션으로 페이지가 바뀌어도 녹화는 이어진다. 새 페이지에서도 오버레이가 다시 준비되고, 펜이 ON 상태였다면 그 상태가 복원돼야 한다(기존 로그 레코더 sentinel 재발행과 동일하게 재주입·재표시).
- **이미 열려 있던 탭 녹화**: 확장 설치·리로드 전부터 열려 있던 탭은 콘텐츠 스크립트가 아직 없을 수 있다 → 녹화 시작 시 programmatic 주입으로 자가복구(picker의 `ensureContentScript` 패턴).
- **화면 녹화로 다른 창/모니터를 공유**: 오버레이는 대상 탭 페이지 위에만 그려지므로, 녹화 대상이 그 탭이 아니면 영상에 안 잡힐 수 있다. 정상 동작이며 안내 대상 아님.
- **그리기 모드 ON 상태로 녹화 종료**: 오버레이·리스너·타이머 모두 정리되어야 한다(리크 방지).
- **펜 ON 중 그린 포인터 이벤트가 액션 로그에 잡히면 안 됨**: action-recorder가 어노테이션 오버레이 이벤트를 자기 UI로 간주해 제외해야 한다(오염 방지).

## 성공 기준

- 탭 녹화·화면 녹화 중 사이드패널 펜 토글 → 페이지 드래그로 선이 그려지고, 약 3초 뒤 페이드아웃된다.
- 그린 선이 저장된 녹화 영상(재생 시)에 보인다.
- 펜 OFF 상태에서는 페이지 버튼·링크가 정상 클릭된다(오버레이 pass-through).
- 그리기 조작이 액션 로그(action-recorder)에 잡히지 않는다.
- 녹화 종료 후 페이지에 어노테이션 DOM·리스너·타이머가 남지 않는다.
- 새 권한 요청 프롬프트가 뜨지 않는다(manifest permissions 무변경).
