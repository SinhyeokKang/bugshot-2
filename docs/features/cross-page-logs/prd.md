# Cross-page 로그 누적 + Replay 30초 트림

## 배경

30s Replay(`fa16efc`)는 직전 30초 화면을 mp4로 첨부한다. 이 프레임 버퍼는 **페이지를 가로질러**(네비게이션 무관) 롤링 30초로 쌓인다 — `docs/features/30s-replay/design.md` 8번 항목에 "버퍼에 이전·새 페이지 프레임이 섞이는 건 의도된 동작"으로 명시돼 있다.

반면 첨부되는 네트워크·콘솔 로그는 페이지 MAIN world 버퍼에 살아서 **풀 네비게이션 시 파괴**되고, `useBackgroundRecorder`가 idle 표준대기 중 네비게이션마다 로그를 리셋한다(페이지 단위). 결과적으로 mp4와 첨부 로그의 시간·페이지 범위가 어긋난다.

예: 8초 전 페이지 A→B로 이동 후 replay 캡처 시 — mp4는 "A 22초 + B 8초"인데, 첨부 로그는 B의 것만 담긴다. mp4 앞 22초(페이지 A)에 대응하는 로그가 없다.

## 목표

- 네트워크·콘솔 로그를 **페이지를 가로질러 누적**한다 (네비게이션 시 리셋하지 않음).
- 떠나는 페이지의 로그 꼬리를 **유실 없이** 누적기에 보존한다.
- **Replay 캡처 시** 첨부 로그를 **mp4 프레임 버퍼의 실제 커버 구간**으로 트림해 정합시킨다. 고정 `now - 30000`이 아니라 **캡처 시점 버퍼의 가장 오래된 프레임 timestamp(`frames[0].timestamp`) ~ 캡처 시점**을 cutoff로 쓴다 — 프레임 버퍼는 개수 cap(60)·시간 cap(30s) 이중 제한 + idle·active 게이팅으로 실제 길이가 30초보다 짧을 수 있으므로, 버퍼가 담은 만큼에 로그를 정확히 맞춘다.
- 사용자가 누적된 로그를 수동으로 비울 수 있는 **[Clear Log] 액션**을 제공한다.

## 모드별 로그 동작

누적기는 단일 공유(editor-store `networkLog`/`consoleLog`). 각 캡처 모드의 첨부 정책:

| 모드 | 로그 동작 |
|---|---|
| idle 표준대기 | cross-page 누적 (네비게이션에도 리셋 안 함) |
| **element** | 누적기 그대로 첨부 (cross-page, 트림 없음) |
| **screenshot** | 누적기 그대로 첨부 (cross-page, 트림 없음) — `startPicking` 시작 시 로그 리셋하지 않고 보존 |
| **freeform** | 누적기 그대로 첨부 (cross-page) — `startFreeform`도 보존 |
| **video 녹화** | **예외**: 녹화 시작 시 로그 클리어, 녹화 구간만 누적 (기존 동작 유지) |
| **replay** | 누적기를 **프레임 버퍼 커버 구간**(`frames[0].timestamp` ~ 캡처 시점)으로 트림해 첨부 |

## 비목표 (Non-goals)

- element/screenshot/freeform 첨부 로그에 트림을 적용하지 않는다 — cross-page 누적 전체를 첨부한다.
- video 녹화 모드를 cross-page 누적에 편입하지 않는다 — 녹화 시작부터 새로 쌓는 현행 동작을 유지한다.
- 로그와 mp4 프레임의 **프레임 단위 정밀 동기화**(타임라인 스크럽 연동 등)는 하지 않는다. 시간 창(window) 정합까지만.
- 누적기에 별도 byte-size 메모리 cap을 새로 도입하지 않는다 (엔트리 body는 MAIN 레코더에서 이미 cap됨, 개수 cap만 적용).
- Clear Log는 mp4 프레임 버퍼를 비우지 않는다 (로그 전용).

## 사용자 시나리오

### S1. Replay 캡처 — 페이지를 가로지른 버그
1. 사용자가 페이지 A에서 버그 재현을 시작, 도중 페이지 B로 이동.
2. B에서 30s Replay 트리거.
3. mp4 = 버퍼가 담은 구간(A 후반 + B). 첨부 로그 = **mp4와 동일한 구간**(버퍼 최오래 프레임 ~ 캡처 시점)의 네트워크·콘솔 엔트리. A에서 떠날 때의 로그도 포함된다.

### S2. 비-replay 캡처 — cross-page 누적 전체
1. 사용자가 A→B→C 이동하며 재현, C에서 element/screenshot 캡처.
2. 첨부 로그 = A+B+C 누적 전체(트림 없음).

### S3. 수동 Clear 후 재현
1. 표준대기 중 로그가 계속 쌓인 상태.
2. 사용자가 로그 탭(Console 또는 Network 서브탭) footer 좌측 **[Clear Log]** 클릭.
3. 해당 서브탭의 로그(누적기 + pending + MAIN world 버퍼)가 비워져 0부터 다시 쌓인다. mp4 프레임 버퍼는 유지된다.

### S4. 세션 경계
1. 이슈 제출/취소 후 idle 복귀.
2. 누적기가 리셋된다(기존 동작 유지) — 다음 버그 세션이 이전 세션 로그를 물려받지 않는다.

## 성공 기준

- Replay 캡처 시 첨부된 네트워크·콘솔 로그의 모든 엔트리 timestamp가 `[버퍼 최오래 프레임 timestamp, captureTime]` 범위 안에 든다 (mp4 커버 구간과 일치).
- 페이지 A에서 B로 풀 네비게이션 후 캡처 시, A에서 발생한(30초 창 내) 엔트리가 첨부 로그에 존재한다.
- 비-replay 모드 캡처 시 누적된 cross-page 로그 전체가 첨부된다.
- [Clear Log] 클릭 후 즉시 해당 로그 표시가 비고, 이후 새 요청만 다시 쌓인다.
- 동일 엔트리가 여러 번 sync돼도 중복 없이 1건으로 유지된다(id dedup).
- `pnpm test` 통과 — 머지/트림 순수 함수 단위 테스트 포함.
