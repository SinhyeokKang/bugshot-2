# 연동 탭 UX 개편

## 배경

현재 "연동" 탭은 상단에 플랫폼별 하위 탭(Jira / GitHub / Linear / Notion)을 두고, 각 플랫폼 탭이 "연결 안 됨 → 온보딩 화면" 또는 "연결됨 → 연결 정보 + 설정"을 보여준다. 이 구조는 다음 문제가 있다.

- **연결된 플랫폼이 흩어진다**: 여러 플랫폼을 연결해도 한 화면에서 한눈에 볼 수 없고 탭을 일일이 옮겨야 한다.
- **온보딩이 플랫폼 탭에 묶여 있다**: 새 플랫폼을 연결하려면 해당 플랫폼 탭으로 이동해야 하고, OAuth/토큰 선택이 빈 화면(centered onboarding)에 섞여 있어 위계가 약하다.
- **다른 탭과 레이아웃이 다르다**: 디버그·이슈작성 탭은 `Section` 기반 collapse 구조인데 연동 탭만 플랫폼 탭 구조라 일관성이 떨어진다.

## 목표

- 연동 탭을 **"내 연동"**(연결된 플랫폼 관리)과 **"플랫폼 추가"**(신규 연결) 두 하위 탭으로 재구성한다.
- "내 연동" 탭은 다른 탭과 동일한 `Section` collapse 구조를 따르고, 연결된 플랫폼마다 [연결 정보 카드 + 설정]을 한 collapsible 섹션에 담는다.
- "플랫폼 추가" 탭은 이슈 작성 idle 레이아웃을 재사용해, 플랫폼별 연결 버튼을 1열 수직으로 배열한다.
- 플랫폼 연결 시 OAuth/토큰 방식 선택을 명시적 컨펌 단계로 분리하되, 기존 토큰 입력 다이얼로그·검증 로직은 그대로 재사용한다.

## 비목표 (Non-goals)

- 인증 로직(OAuth 플로우, 토큰 교환·검증 메시지, 백그라운드 핸들러) 변경 없음. 단 **Jira 다중 사이트 선택은 인증 로직(BG 핸들러·토큰 교환)은 불변이고 사이트 선택 *표현*만 인라인 패널 → 다이얼로그로 이동**(UI 재배치에 한함).
- 플랫폼별 설정 필드(프로젝트/리포/팀/DB 콤보박스 등)의 동작·구성 변경 없음.
- 새 플랫폼 추가나 새 인증 수단 추가 없음.
- 연결 정보 카드(`ConnectedBadge`, `Summary`)의 표시 항목 변경 없음.
- 권한·manifest·외부 API·env 변경 없음.
- 상위 앱 탭(debug / issue-list / integrations / settings) 구조·자동진입 로직 변경 없음. 본 개편은 **연동 탭 내부**만 재구성한다.

## 사용자 시나리오

### S1. 연결된 플랫폼이 없는 사용자
1. 확장 최초 사용 시 App이 연동 탭으로 자동 진입(기존 동작).
2. 연결된 플랫폼이 없으므로 **"플랫폼 추가"** 하위 탭이 기본 선택된다.
3. 중앙에 아이콘 + 안내 + 1열 버튼 목록(Jira / GitHub / Linear / Notion 연결)이 보인다.
4. "Jira 연결" 클릭 → OAuth/토큰 선택 컨펌 → 선택한 방식으로 연결.
5. 연결 성공 시 자동으로 **"내 연동"** 하위 탭으로 이동해 방금 연결한 Jira 섹션이 보인다.

### S2. 연결된 플랫폼이 있는 사용자
1. 상위 "연동" 탭 클릭 시 **"내 연동"** 하위 탭으로 진입한다.
2. 연결된 플랫폼마다 collapsible 섹션이 세로로 나열된다. 섹션 헤더에 플랫폼 이름 + 연동 해제 버튼 + 아코디언 토글이 있다.
3. 섹션을 펼치면 [연결 정보 카드][설정 필드]가 보인다. 접으면 헤더만 남는다.
4. 2개 이상 연결돼 있으면 하단 footer에 "전체 연결 해제" 버튼이 있다.
5. 섹션별 해제 또는 전체 해제로 연결이 0개가 되면 자동으로 **"플랫폼 추가"** 하위 탭으로 전환된다(빈 "내 연동" 화면에 머무르지 않음).

### S3. 추가 연결 / 단일 수단
1. "내 연동" 또는 "플랫폼 추가" 하위 탭에서 "플랫폼 추가"로 이동.
2. 이미 연결된 플랫폼 버튼은 disabled로 표시된다(목록에는 항상 4개 모두 노출).
3. OAuth env가 미설정된 플랫폼은 컨펌 없이 바로 토큰 입력 다이얼로그가 열린다(가능한 수단이 1개라서).
4. 두 수단 모두 가능하면 컨펌 창에서 OAuth/토큰 중 선택.

### S4. 엣지 — Jira 다중 사이트 / 프로젝트 미선택
1. Jira OAuth 성공 시 접근 가능한 사이트가 2개 이상이면 **사이트 선택 다이얼로그**가 열린다(기존 인라인 패널 대체).
2. 사이트 선택 후, Jira 계정에 `projectKey`가 없으면 기존 **프로젝트 선택 다이얼로그(SetupDialog)**가 "내 연동" 탭에서 자동으로 열린다. 취소 시 Jira 연동이 해제된다(기존 동작 유지).

## 성공 기준

- 연동 탭에 "내 연동" / "플랫폼 추가" 두 하위 탭이 존재한다.
- "내 연동": 연결된 플랫폼이 각각 collapsible 섹션으로 나열되고, 섹션별 연동 해제 + (2개 이상 시) 전체 해제가 동작한다. 연결이 없으면 빈 상태 + "플랫폼 추가"로 가는 CTA가 보인다.
- "플랫폼 추가": 4개 플랫폼 연결 버튼이 1열로 배열되고, 연결된 것은 disabled. 클릭 시 OAuth/토큰 선택(수단 1개면 생략) → 연결 완료.
- 상위 연동 탭 진입 시 연결 유무에 따라 올바른 하위 탭으로 라우팅된다.
- Jira 다중 사이트 선택, 프로젝트 미선택 강제 다이얼로그가 정상 동작한다.
- 연결이 0개가 되면(섹션별/전체 해제) "플랫폼 추가" 탭으로 자동 전환된다.
- 기존 인증·검증·설정 동작에 회귀가 없다. 회귀 검증은 인증·OAuth·컴포넌트가 순수함수 테스트 영역 밖이므로 **tasks.md의 수동 테스트 체크리스트**로 확인한다. 순수 함수(하위 탭 결정·수단 판정)와 onConnected 콜백 타이밍은 자동 테스트로 커버하고 `pnpm test` 통과.
