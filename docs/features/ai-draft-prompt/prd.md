# AI 드래프트 버그 설명 프롬프트

## 배경

현재 AI 초안 생성은 요소 모드에서만 동작한다. 요소의 CSS selector, tag name, style diff, design token 등 풍부한 메타데이터가 있기 때문이다. 스크린샷 모드는 URL + 페이지 제목 외에 AI에 줄 컨텍스트가 없어 버튼이 숨김 처리되어 있고, 비디오 모드는 에러 로그 기반으로 동작하지만 "사용자가 뭘 하려다 문제가 생겼는지"라는 핵심 맥락이 빠져 있다.

## 목표

1. 스크린샷·비디오 모드에서 AI 초안 버튼 클릭 시 **버그 설명 프롬프트 다이얼로그**를 띄워 사용자의 자연어 설명을 입력받는다.
2. 스크린샷 모드에서는 캡처 이미지를 **멀티모달**(base64)로 AI에 함께 전송해 시각적 맥락을 제공한다.
3. 다이얼로그는 **멀티턴 대화**를 지원해 초안이 불만족스러우면 추가 지시로 개선할 수 있다.
4. `generate()` 인터페이스에 이미지 지원을 추가해 향후 다른 기능에서도 멀티모달을 활용할 수 있게 한다.

## 비목표 (Non-goals)

- 요소 모드의 기존 AI 초안 흐름 변경 (즉시 생성 유지, 다이얼로그 없음)
- 비디오 모드에서 썸네일 이미지를 AI에 전송하는 것 (텍스트 컨텍스트만)
- Chrome AI(LanguageModel API)의 멀티모달 지원 (현재 텍스트 전용 — 이미지 파라미터를 무시)
- AI 스타일링 다이얼로그(AiStylingDialog) 변경

## 사용자 시나리오

### 시나리오 1: 스크린샷 모드 AI 초안

1. 사용자가 스크린샷 캡처 완료 → drafting 단계 진입
2. 하단 AI 초안 버튼 노출 (기존에는 숨겨져 있었음)
3. 버튼 클릭 → 버그 설명 프롬프트 다이얼로그 열림
4. 사용자가 "로그인 버튼 클릭 시 아무 반응 없음" 등 버그 설명 입력
5. 제출 → 다이얼로그 닫힘 → 로딩 오버레이(shimmer) 표시
6. AI가 스크린샷 이미지 + 사용자 설명 + URL/페이지 제목을 기반으로 초안 생성
7. 초안이 title + 각 섹션에 채워짐
8. 불만족 시 AI 버튼 재클릭 → 다이얼로그 재오픈 → "재현 단계를 더 구체적으로" 등 추가 지시 → 세션 유지 상태에서 초안 갱신

### 시나리오 2: 비디오 모드 AI 초안

1. 사용자가 비디오 녹화 완료 → drafting 단계 진입
2. AI 초안 버튼 클릭 → 버그 설명 프롬프트 다이얼로그 열림
3. 사용자가 버그 설명 입력 → 제출
4. AI가 사용자 설명 + 에러 로그(있으면) + URL/페이지 제목을 기반으로 초안 생성
5. 이후 흐름은 시나리오 1과 동일

### 시나리오 3: 다이얼로그 취소

1. AI 초안 버튼 클릭 → 다이얼로그 열림
2. 취소 클릭 (또는 ESC) → 다이얼로그 닫힘
3. 아무 일도 일어나지 않음. 사용자가 수동으로 초안 작성.

### 엣지 케이스

- AI 미설정 상태: 버튼 자체가 노출되지 않음 (기존 `aiStatus === "available"` 조건 유지)
- 빈 입력 제출: 제출 버튼 비활성화 (AiStylingDialog와 동일)
- API 에러 / 429: 토스트로 에러 메시지, 세션은 유지 (재시도 가능)
- 응답 파싱 실패: 에러 메시지 노출, 세션 유지

## 성공 기준

- [ ] 스크린샷 모드에서 AI 초안 버튼이 노출되고, 클릭 시 다이얼로그가 열린다
- [ ] 비디오 모드에서 AI 초안 버튼 클릭 시 다이얼로그가 열린다 (기존: 즉시 생성)
- [ ] 다이얼로그에서 설명 입력 후 제출하면 AI 초안이 생성된다
- [ ] 스크린샷 모드에서는 캡처 이미지가 AI에 멀티모달로 전송된다
- [ ] 멀티턴 대화로 초안 개선이 가능하다
- [ ] 취소 시 아무 동작 없이 다이얼로그만 닫힌다
- [ ] 요소 모드의 기존 동작에 변화 없다
- [ ] ko/en 다국어 지원
