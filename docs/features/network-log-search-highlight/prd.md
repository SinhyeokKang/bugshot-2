# 네트워크 로그 검색어 하이라이트

## 배경

로그 탭(console/network/action)은 이미 텍스트 검색을 제공하지만, 매칭 결과를 **필터링만** 한다 — 검색어에 걸린 행만 남기고, 정작 **어느 문구가 왜 매칭됐는지**는 표시하지 않는다. 특히 네트워크 로그는 `requestMatchesQuery`가 URL·요청/응답 헤더·요청/응답 바디까지 훑어 매칭하므로, 검색에 걸린 요청을 열어도 헤더·페이로드 어디에 검색어가 있는지 눈으로 다시 찾아야 한다.

검색어에 배경색 하이라이트를 입히면(브라우저 기본 검색·문서 검색 UI의 관례) 매칭 위치를 즉시 파악할 수 있다.

## 목표

- 네트워크 로그 **상세 패널**에서 현재 검색어와 일치하는 문구에 배경색(앰버) 하이라이트를 표시한다.
- 하이라이트 대상: 상세 패널의 세 영역 전부
  - **headers 탭**: General 섹션(URL·method·status·time·contentType) + 응답 헤더 + 요청 헤더(key/value)
  - **request 탭**: 요청 바디(payload) — plain text `<pre>` 및 JSON 트리
  - **response 탭**: 응답 바디 — plain text `<pre>` 및 JSON 트리
- 검색어가 비어 있으면 기존과 **완전히 동일한** 렌더(하이라이트 없음, 동작·성능 변화 0).
- 대소문자 무시·부분 일치 — 기존 네트워크 검색(`requestMatchesQuery`, 소문자 substring)과 동일 규칙.

## 비목표 (Non-goals)

- **콘솔·액션 로그 하이라이트 제외.** 이 둘은 레벨(error/warn/info)·종류별 **행 배경색**을 이미 쓰고 있어, 하이라이트 배경이 겹치면 구분이 무너진다.
- **접힌(collapsed) 네트워크 로그 행의 URL 하이라이트 제외.** 상세를 펼쳤을 때만 하이라이트한다.
- **WebSocket `messages` 탭 하이라이트 제외.** 요청은 헤더·페이로드·리스폰스로 한정. (`FrameBody`가 동일 프리미티브를 쓰므로 후속 확장은 저비용이나 이번 스코프 아님.)
- **매치 네비게이션(다음/이전 이동·매치 개수 표시) 제외.** 검색이 이미 비매칭 요청을 목록에서 걸러내므로 단순 배경 하이라이트로 충분.
- 검색 로직·매칭 대상 필드 변경 없음(하이라이트는 순수 표시 계층).

## 사용자 시나리오

1. 사용자가 네트워크 로그 탭에서 검색창에 `token`을 입력한다.
2. 목록이 `token`을 URL·헤더·바디 어딘가에 포함한 요청만 남긴다(기존 동작).
3. 사용자가 그중 한 요청을 클릭해 상세를 연다.
4. **headers 탭**의 응답 헤더 `Authorization: Bearer …token…`에서 `token` 문구가 앰버 배경으로 강조된다.
5. **response 탭**으로 이동하면 JSON 응답 트리에서 `"access_token"` 키·`"…token…"` 문자열 값의 매칭 부분이 앰버 배경으로 강조된다.
6. 검색창을 비우면(X 클릭) 하이라이트가 사라지고 상세는 원래 렌더로 돌아온다.

### 엣지 케이스

- **URL로만 매칭된 요청**: 상세를 열면 General 섹션의 URL에만 하이라이트가 뜨고 헤더·바디에는 없다(정상 — 그 필드엔 검색어가 없음).
- **정규식 특수문자 검색어**(`api.v2(beta)` 등): 리터럴로 취급(정규식 이스케이프).
- **JSON 문자열 값이 300자 초과로 잘린 경우**(`StringRow`의 truncate): 잘린 앞 150자 범위 밖 매칭은 "전체 보기"를 눌러야 보인다(기존 truncate 동작 유지, 하이라이트가 truncate를 바꾸지 않음).
- **마스킹된 헤더 값**(`***`로 시작): 하이라이트 미적용(원래 italic muted 렌더 유지).

## 성공 기준

- 네트워크 상세 3개 탭(headers/request/response)에서 검색어 매칭 문구가 앰버 배경으로 표시된다.
- JSON 바디(트리 렌더)와 non-JSON 바디(`<pre>`) 양쪽에서 하이라이트가 동작한다.
- 검색어가 비면 상세 렌더가 검색 전과 픽셀 단위로 동일하다.
- 콘솔·액션 로그, 접힌 네트워크 행, WS messages 탭에는 하이라이트가 나타나지 않는다.
- `splitHighlight` 순수 함수 단위 테스트가 통과한다(대소문자·이스케이프·다중 매칭·빈 쿼리).
