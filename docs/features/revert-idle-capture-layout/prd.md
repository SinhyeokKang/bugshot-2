# Idle 캡처 진입 레이아웃 원복 (1x2x2)

## 배경
`recording-mode-setting` 기능이 idle 캡처 진입 화면(`EmptyState`)을 2-1-3 레이아웃으로 재구성했다:
- Row1 `[요소 편집][요소 캡처]`
- Row2 `[범위 캡처]`
- Row3 `[녹화][30초 리플레이][⚙ 녹화 설정]` (segmented)

⚙ 버튼이 인라인 "녹화 설정 다이얼로그"(녹화 모드 Tabs + 리플레이 토글)를 띄우고, 비활성 리플레이 버튼도 같은 다이얼로그를 연다.

이 재구성으로 요소 편집이 outline 버튼으로 강등되고, 진입 화면에 설정 진입점(⚙)·다이얼로그가 추가돼 화면이 복잡해졌다. 요소 편집을 다시 1차 액션(primary)으로 끌어올리고 진입 화면을 단순하게 되돌린다.

> **baseline**: 원복 기준은 `recording-mode-setting` 도입 *전* 원본(`b26dd01^`)이다. 그 원본도 요소 편집이 primary(전체폭 단독)였으므로 primary 승격은 신규 결정이 아니라 진짜 원복이다. 단 Row3만 원본(`[탭 녹화][화면 녹화]` 2버튼 + 별도 리플레이 행)과 다르게 단일 record 버튼 + 리플레이로 둔다(아래 비목표 참조).

## 목표
- idle 캡처 진입 레이아웃을 **1x2x2 3행**으로 되돌린다:
  - Row1 `[요소 스타일 편집]` — primary 버튼, 전체폭 단독 (실제로는 `max-w-[336px]` 컨테이너 안의 full-width)
  - Row2 `[요소 캡처][범위 캡처]`
  - Row3 `[녹화][30초 리플레이]`
  - footer(`[가이드] ... [이슈 초안 작성]`)는 변경 없이 유지.
- Row1 레이블을 `issue.mode.element` 키 값 **"요소 스타일 편집"/"Edit element style"로 변경**(기존 "요소 편집"/"Edit element"에서). 키 추가·삭제는 아니고 기존 키 값만 ko/en 동시 수정.
- `⚙ 녹화 설정` 버튼 제거.
- 녹화 설정 다이얼로그(`RecordingSettingsDialog`) 제거.
- 비활성(replayEnabled=off) 30초 리플레이 버튼 클릭 시 **설정 탭 > 이슈 sub-tab**으로 이동(기능 전 원본 동작 원복).
- 설정된 녹화 모드(tab/screen)에 따른 **녹화 버튼 아이콘·레이블 분기는 유지**(단일 record 버튼, `recordModeMeta`).

## 비목표 (Non-goals)
- 녹화 모드 자체(`recordingMode` 상태·설정 탭 `RecordingSettingsCard`)는 제거하지 않는다 — 설정 탭에서 계속 변경 가능.
- 단일 record 버튼을 video/screen-record 2개 분리 버튼으로 되돌리지 않는다(기능 전 원본과 다른 점). 모드 분기 단일 버튼 유지.
- i18n 키 **추가·삭제** 없음(`issue.mode.element` 값만 수정, 그 외 키 재사용).
- store 마이그레이션 변경 없음(`recordingMode` 필드 그대로).
- idle 화면에 **녹화 모드(tab/screen) 전환 진입점을 두지 않는다**. ⚙ 제거로 모드 변경은 설정 탭 단일 경로가 된다. 기능 전 원본도 idle에 모드 전환 단서가 없었고(모드 개념 자체가 없었음) 모드 전환은 드문 동작이라는 가정 하에 이 마찰 증가를 수용한다.

## 사용자 시나리오
1. 지원 페이지에서 디버그 > 이슈 작성 진입 → idle 캡처 화면.
2. 최상단 **요소 스타일 편집** primary 버튼이 가장 눈에 띈다. 클릭 → picker 시작.
3. 둘째 행에서 요소 캡처 / 범위 캡처 선택 가능.
4. 셋째 행 **녹화** 버튼은 설정된 녹화 모드 아이콘·레이블로 표시(tab=AppWindow/"탭 녹화", screen=MonitorPlay/"화면 녹화"). 클릭 → 해당 모드로 녹화 시작.
5. **30초 리플레이** 버튼이 녹화 옆에 균등 너비로 표시.
   - 엣지: 리플레이가 off면 버튼이 비활성처럼 보이고, 클릭하면 설정 탭 > 이슈 sub-tab으로 이동(거기서 토글).
   - 엣지: 녹화 모드를 설정 탭에서 바꾸면 idle로 돌아왔을 때 녹화 버튼 아이콘·레이블이 반영된다.

## 성공 기준
- idle 화면이 1x2x2(3행) 구조로 렌더되고, Row1이 primary 요소 스타일 편집 단독 행이다.
- **요소 스타일 편집이 시각적으로 가장 강조된다**(유일한 primary variant). 진입 화면에서 설정 다이얼로그·⚙ 버튼이 사라져 노출 컨트롤 수가 줄어든다.
- Row1 레이블이 ko "요소 스타일 편집" / en "Edit element style"로 렌더된다.
- `mode-record-settings`(⚙) 버튼과 녹화 설정 다이얼로그가 DOM에 더는 존재하지 않는다.
- 녹화 버튼이 `recordingMode`에 따라 AppWindow/MonitorPlay 아이콘으로 분기된다.
- 비활성 리플레이 클릭 시 설정 탭으로 이동한다(다이얼로그 안 뜸).
- `pnpm typecheck` 통과, 관련 e2e green.
