# Draft 텍스트 필드 수정

## 배경

draft 이슈의 상세 다이얼로그(`DraftDetailDialog`)는 현재 완전 읽기 전용이다. 제목·발생 현상·기대 결과 같은 텍스트를 잘못 적었거나 보강하고 싶으면 수정할 방법이 없다. 유일한 수정 경로는 라이브 편집기(`DraftingPanel`)인데, 이건 캡처 직후의 미확정 draft만 다루고 이미 리스트에 저장된 draft 이슈로는 되돌아갈 수 없다. 결과적으로 사용자는 오탈자 하나를 고치려고 이슈를 삭제하고 처음부터 다시 캡처해야 한다.

## 목표

- 저장된 draft 이슈의 **텍스트 데이터(제목 + 본문 섹션)를 상세 다이얼로그에서 직접 수정**할 수 있다.
- 수정 결과가 `issues-store`에 영속되어 다이얼로그·리스트·이후 제출 본문에 즉시 반영된다.
- 수정 UI는 라이브 편집기와 **동일한 입력 위젯**(제목 Input, 문단 Tiptap, 재현 절차 번호목록 에디터)을 재사용해 마크다운·인라인 이미지·번호목록 정합이 깨지지 않는다.

## 비목표 (Non-goals)

- `DraftDetailDialog` 본체를 인라인 편집(readOnly 해제)으로 바꾸지 않는다. 본체는 지금처럼 읽기 전용이고, 편집은 별도 다이얼로그에서만 일어난다.
- **재현 환경(reproduction environment) 행 편집 제외.** OS·Browser·Page·DOM·Viewport·Captured 등 자동 파생 메타와 `draft.environment` 사용자 입력 행 모두 이번 스코프 밖.
- 미디어(스크린샷/영상)·스타일 변경 표·로그·첨부 섹션 수정 제외.
- 수정 다이얼로그에서 **새 이미지 삽입 버튼(카메라 영역캡처·이미지 첨부 파일 선택) 제외.** Tiptap 에디터 영역에 대한 **드래그앤드롭/붙여넣기 이미지 삽입만** 허용(Tiptap 내장 동작 그대로).
- submitted(제출 완료·Slack 승격 보존) 이슈 수정 제외 — 이번 기능은 `status: "draft"` 이슈 전용. submitted 상세는 [수정] 버튼도, 빈 섹션 신규 노출도 없다(기존 렌더 그대로).
- **비활성(disabled) 섹션 상세 노출 제외.** `notes`(비고)처럼 `enabled: false`인 섹션은 상세에 나타나지 않으므로 [수정] 버튼도 없다. 이런 섹션을 채우려면 먼저 설정에서 해당 섹션을 활성화해야 한다(상세→편집 경로만으로는 추가 불가).

## 사용자 시나리오

1. 사용자가 이슈 리스트에서 draft 이슈의 [자세히]를 눌러 `DraftDetailDialog`를 연다.
2. 제목과 **enabled**인 본문 섹션(발생 현상=description·재현 절차=stepsToReproduce·기대 결과=expectedResult 등) 각 헤더에 **[수정] Pencil 아이콘 버튼**이 보인다.
3. 예: "발생 현상" 섹션의 [수정] 클릭 → 편집 다이얼로그가 뜨고, 그 섹션의 현재 내용이 라이브 편집기와 동일한 Tiptap 에디터에 채워져 있다.
4. 텍스트를 수정한다(이미지가 필요하면 에디터 영역에 파일을 드래그앤드롭하거나 붙여넣는다) → [저장].
5. 다이얼로그가 닫히고 상세 화면의 해당 섹션이 즉시 새 내용으로 갱신된다. 이후 제출 시 이 내용이 본문에 반영된다.

### 엣지 케이스

- **빈 섹션 수정(draft 전용)**: enabled인데 내용이 비어 있는 섹션도(현재는 상세에서 아예 숨겨짐) **`status: "draft"`일 때만** [수정] 버튼과 함께 노출해 새로 채울 수 있다. submitted 상세에서는 이 완화가 적용되지 않아 빈 섹션이 새로 노출되지 않는다.
- **제목 수정 시 리스트 동기화**: 리스트 행(`IssueRow`)과 검색 필터는 최상위 `issue.title`을 쓰고 상세는 `issue.draft.title`을 쓴다. 생성 시 둘은 동일하므로, 제목 수정은 **두 값을 함께 갱신**해야 리스트/검색과 어긋나지 않는다.
- **재현 절차(orderedList)**: 줄 단위 번호목록 에디터로 편집하며 저장 값은 개행 결합 문자열(기존 포맷 유지).
- **빈 제목 저장 차단**: 편집 다이얼로그에서 제목이 빈 문자열이면 [저장]을 비활성화한다(라이브 편집기의 `titleMissing` 가드와 일관). 본문 섹션은 빈 값 저장을 허용(내용 clear 가능).
- **취소**: [취소]하거나 다이얼로그를 닫으면 원본이 그대로 유지되고 아무것도 저장되지 않는다.

## 성공 기준

- draft 상세에서 제목·enabled 본문 섹션을 각각 수정→저장하면 상세·리스트·검색에 반영되고, **수정 후 재제출 시 이슈 본문 마크다운에 새 텍스트가 포함**된다.
- 제목 수정 후 리스트 행 제목과 검색 결과가 새 제목을 반영한다.
- 인라인 이미지가 포함된 섹션을 수정해도 기존 이미지가 유지되고, DND로 새 이미지를 추가할 수 있다.
- 재현 환경·미디어·로그·첨부 섹션에는 [수정] 버튼이 나타나지 않는다. submitted 상세에는 [수정] 버튼이 없다.
- 편집 다이얼로그에서 [취소]/닫기 시 원본이 그대로 유지된다(미저장).
- 빈 제목은 저장할 수 없다([저장] 비활성).
- 수정 저장 후 `updatedAt`이 갱신된다.
