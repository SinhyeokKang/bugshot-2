# 이슈 파일 첨부 (Issue File Attachment)

## 배경

현재 이슈에 붙는 파일은 확장이 자동 생성한 것뿐이다 — 캡처(스크린샷·비디오·요소 before/after)와 로그(`logs.html`). 사용자가 손에 들고 있는 임의 파일(디자인 PDF, 사양서, 별도 로그/덤프, 외부에서 받은 zip, 추가 스크린샷 등)을 이슈에 함께 올릴 경로가 없다. 지금은 이슈 생성 후 플랫폼 웹에서 따로 첨부해야 한다.

## 목표

- 이슈 작성(drafting) 화면에서 **버튼으로 로컬 파일을 골라 첨부**할 수 있다. 파일 종류 제한 없음(모든 MIME).
- 첨부한 파일이 6개 플랫폼(Jira·GitHub·Linear·Notion·GitLab·Asana) 모두에서 생성된 이슈에 올라간다.
- 첨부는 **이슈 본문 설정의 토글로 ON/OFF** 한다. **기본값 OFF**(권장 동작이 아니므로 — 캡처·로그가 1차 수단). OFF일 때 첨부 UI 자체가 노출되지 않는다.
- 첨부 파일은 draft에 영속되어 사이드패널을 닫았다 열거나 다른 탭에 갔다 와도 유지된다(캡처 로그와 동일한 영속 보장).
- **개수 상한**(파일 N개)과 **플랫폼 용량 한도 경고**(Notion 5MiB, GitLab 10MB 등)를 사전에 제공한다.

## 비목표 (Non-goals)

- **본문 인라인 임베드 안 함**. 첨부 이미지를 본문 중간에 미리보기로 렌더하지 않는다. 플랫폼 네이티브 첨부 영역(Jira/Asana/Notion) 또는 본문 하단 링크 목록(GitHub/GitLab/Linear)으로만 노출.
- 드래그앤드롭 진입 없음. 버튼(파일 선택 다이얼로그)만.
- 첨부 파일의 미리보기·편집·주석(annotation) 없음.
- 클립보드 붙여넣기로 첨부 추가 없음(기존 본문 inline 이미지 붙여넣기와 별개 기능).
- 업로드 진행률 바·재시도 UI 없음. 기존 캡처 업로드와 동일한 성공/부분실패 처리 재사용.
- 파일 종류별 차등 처리 없음(이미지든 zip이든 동일하게 첨부로만).

## 사용자 시나리오

### 기본 플로우
1. 설정 → 이슈 → 본문 구성에서 **"파일 첨부"** 토글을 켠다(기본 OFF).
2. 요소/스크린샷/비디오/자유형 중 한 모드로 캡처해 drafting 화면에 진입한다.
3. drafting 화면의 첨부 섹션에서 **"파일 첨부"** 버튼을 누른다 → OS 파일 선택 다이얼로그(다중 선택 가능).
4. 고른 파일들이 리스트로 표시된다(파일명 + 크기 + 삭제 버튼).
5. 이슈를 제출하면 캡처·로그와 함께 첨부 파일이 플랫폼에 업로드된다.
6. Jira/Asana/Notion은 네이티브 첨부 영역에, GitHub/GitLab/Linear는 본문 하단 "Attachments" 링크 목록에 나타난다.

### 엣지 케이스
- **개수 상한 초과**: 상한(예: 10개)에 도달하면 버튼이 비활성화되고 안내 문구 표시. 다중 선택으로 상한을 넘기면 앞에서부터 상한까지만 받고 나머지는 무시 + 경고.
- **플랫폼 한도 초과**: 현재 선택된 타깃 플랫폼의 파일 단건/합계 한도를 넘는 파일이 있으면 리스트에 경고 배지 + 제출 영역에 경고 문구. 제출은 막지 않음(플랫폼이 거부하면 기존 부분실패 처리로 흡수).
- **세션 복원**: 파일 첨부 후 사이드패널을 닫았다 다시 열면 첨부 리스트가 그대로 복원된다. 비디오와 달리 유실되지 않는다(IndexedDB 영속).
- **토글 OFF로 전환**: 첨부가 있는 상태에서 설정 토글을 끄면 drafting의 첨부 섹션이 숨겨지고, 제출 시 첨부는 업로드되지 않는다(보관된 Blob은 draft 폐기/제출 시 정리). 다시 켜면 보관된 첨부가 다시 보인다.
- **draft 폐기/제출 완료**: 캡처 로그와 동일하게 첨부 Blob도 IndexedDB에서 정리된다(orphan 누적 방지).
- **빈 파일/0바이트**: 그대로 첨부 허용(플랫폼 판단에 위임).

## 성공 기준

- 본문 설정 토글이 OFF면 drafting에 첨부 섹션이 없고, ON이면 나타난다. 기본은 OFF.
- 버튼으로 다중 파일을 골라 리스트에 추가/개별 삭제할 수 있다.
- 6개 플랫폼 각각에서 생성된 실제 이슈에 첨부 파일이 접근 가능한 형태로 존재한다.
- 사이드패널 재오픈/탭 전환 후에도 첨부 리스트가 유지된다.
- 개수 상한과 플랫폼 한도 경고가 동작한다.
- 제출/폐기 후 IndexedDB에 첨부 Blob 잔여물이 남지 않는다(orphan prune 통과).
- 관련 순수 함수 단위 테스트와 `pnpm test` 통과.
