# 본문 구성 순서 변경 (Body Composition Reorder)

## 배경

**Why now (트리거)**: 사용자가 이슈 본문 섹션의 배치 순서를 직접 제어하고 싶다고 요청했다(예: 팀 트래커 템플릿에 맞춰 재현 과정을 앞세우거나, 미디어를 상단에 두기). 현재는 순서를 바꿀 수단이 전혀 없다.

이슈 본문은 여러 섹션(발생 현상·재현 과정·기대 결과·비고)과 자동 블록(미디어·스타일 diff·로그 요약·환경·첨부)으로 구성된다. 섹션 순서는 `DEFAULT_ISSUE_SECTIONS` 배열에 **하드코딩**돼 있고, 사용자는 각 섹션의 사용 여부(`enabled`)만 토글할 수 있다.

미디어/스타일 diff/로그 요약 블록은 `POST_MEDIA_SECTION_IDS` 앵커 규칙(= enabled인 `expectedResult`/`notes` 중 첫 항목 직전, 없으면 말미)으로 배치되며, 이 규칙이 **8개 플랫폼 빌더 + DraftingPanel + DraftDetailDialog + composePreviewLayout까지 11곳에 복제**돼 있다. 순서를 사용자가 바꿀 수 있게 되는 순간 이 고정 앵커는 미디어가 예측 불가하게 튀는 문제를 낳는다(→ 미디어를 순서 리스트의 항목으로 편입해 앵커 규칙 자체를 삭제).

설정 화면 IA에도 어색함이 있다: 토글 하나(`재현 과정 채우기`)만 든 "AI 설정" 섹션이 독립 섹션을 차지하고, 본문에 인라인되지 않는 "파일 첨부"가 본문 구성 섹션에 섞여 있다(reorder 도입 시 "이것도 드래그되나?" 혼란 유발).

## 목표

1. **본문 구성 섹션 순서를 드래그로 변경**할 수 있다. 순서는 변경 즉시 영속된다.
2. 미디어/스타일 diff/로그 요약을 **하나의 순서 조정 가능한 카드("미디어 · 로그")**로 리스트에 편입한다. 이 카드는 **사용 여부 스위치가 없다** — 이유는 "필수라서"가 아니라, **본문 emit이 데이터 기반이라 캡처가 없으면 자동 생략되고, 로그/파일 첨부의 on/off는 이미 drafting 단계 토글(`logsAttach`·`attachmentsEnabled`)로 존재하므로, 설정 카드는 오직 위치만 관장**하기 때문이다.
3. 설정한 순서가 draft 패널·저장 draft 상세·초안·8개 플랫폼 마크다운/ADF/Notion 빌더에 **그대로 상속**된다(현행 단일 출처 구조 유지).
4. 설정 IA 정리:
   - 새 **"기타" 섹션**을 이슈 설정 탭 최하단에 신설. 내부 순서 = `재현 과정 채우기` → `파일 첨부`.
   - 기존 **"AI 설정" 섹션 제거**(`재현 과정 채우기`는 기타로 이전).
   - **파일 첨부**를 본문 구성에서 기타로 이전.
5. **순서 초기화 경로 제공**: 본문 구성 Section 제목 우측에 "기본 순서 복원" 아이콘 버튼(기존 `RotateCcw` reset UX 재사용). 현재 순서가 기본값과 같으면 비활성.
6. **빌더 회귀 봉쇄**: 마이그레이션된 기본 순서에서 8개 빌더의 본문 출력이 변경 전과 **바이트 동일**해야 한다(리팩터 착수 전에 현행 출력을 골든으로 박제 → 리팩터 후 동일성 검증).

## 비목표 (Non-goals)

- **draft별 순서 스냅샷** [영구]: 저장된 draft는 순서를 자체 저장하지 않는다. 항상 현재 전역 설정 순서를 빌드 시점에 파생한다(설정을 바꾸면 기존 저장 draft 출력 순서도 바뀐다). per-draft reorder UI가 없어 스냅샷은 편집 불가능한 값을 박제할 뿐이다. → draft 모델 변경·마이그레이션 없음.
- **명시적 `order` 필드** [영구]: 순서는 `issueSections` 배열의 인덱스로 표현한다. 별도 `order:number` 필드를 두지 않는다(이중 출처 desync 방지).
- **환경(env) 블록 순서 변경** [유예]: env는 항상 본문 최상단 고정. 요청되면 후속에서 동일 메커니즘으로 편입 가능.
- **첨부(attachments) 블록 본문 위치 변경** [영구]: 본문 최하단 고정. 설정의 토글만 기타로 이전.
- **섹션 추가/삭제·라벨 편집** [유예]: 기존 4개 텍스트 섹션 + 미디어 카드만. 커스텀 섹션은 향후 별도 기능.
- **미디어 카드 내부 순서 변경** [유예]: `미디어 → 로그 요약` 고정.

## 사용자 시나리오

### S1. 섹션 순서 변경
1. 설정 → 이슈 설정 → 본문 구성 카드로 이동한다.
2. 각 행 좌측의 드래그 핸들(GripVertical)을 잡고 위아래로 드래그한다. 키보드로도 가능(핸들 focus → Space → 방향키 → Space).
3. 놓는 즉시 순서가 `chrome.storage.local`에 영속된다.
4. 이후 새 이슈 작성 시 draft 패널·프리뷰·제출 본문이 그 순서로 렌더된다.

### S2. 미디어 위치 조정
1. 본문 구성 리스트에서 "미디어 · 로그" 카드(스위치 없음, 핸들만)를 드래그한다.
2. 예: 맨 위로 올리면 캡처 결과물 + 로그 요약이 발생 현상보다 앞에 온다.
3. 캡처 모드가 이 카드에 렌더되는 내용을 결정한다: **element 모드** → 스타일 diff 테이블(+스냅샷), **screenshot/video 모드** → 미디어 블록, **freeform 모드** → 미디어 없음(로그 요약만). 어느 경우든 카드 위치가 그 블록의 본문 위치를 정한다.

### S3. 기타 설정
1. 이슈 설정 탭 최하단 "기타" 섹션에서 `재현 과정 채우기`(AI 자동 채움) 토글과 `파일 첨부` 토글을 조작한다.
2. `재현 과정 채우기`는 `재현 과정` 섹션이 비활성이면 자동 비활성(disabled) — 기존 가드 유지.

### S4. 순서 초기화
1. 순서를 꼬아놓은 사용자가 본문 구성 Section 제목 우측의 복원 버튼을 누른다.
2. 순서·미디어 카드 위치가 기본값으로 되돌아간다. 이미 기본값이면 버튼은 비활성.

### 엣지 케이스
- **캡처 데이터 없음**: 미디어 카드는 리스트에 항상 보이지만, 캡처된 미디어·로그가 없으면 본문에는 아무것도 emit되지 않는다(데이터 기반).
- **freeform 모드**: 미디어 카드 위치에는 미디어 블록이 없고 로그 요약만 온다.
- **발생 현상 비활성**: 미디어 카드는 자체 슬롯이라 발생 현상 토글과 무관하게 자기 위치에 렌더된다.
- **기존 사용자(설정 v8)**: 저장된 `issueSections`에 미디어 엔트리가 없다 → 마이그레이션이 레거시 앵커 위치에 삽입 → 출력 레이아웃 불변.
- **미디어 엔트리 중복/누락(오염 상태)**: 정규화가 "정확히 1개"를 보장(dedupe + 누락 시 backfill)해, 본문에서 미디어가 소실되거나 중복 렌더되지 않는다.

## 성공 기준

- [ ] 본문 구성 5개 항목(발생현상·재현과정·미디어·기대결과·비고)을 드래그 **및 키보드**로 재정렬할 수 있고, 새로고침 후에도 순서가 유지된다.
- [ ] 미디어 카드에는 사용 여부 스위치가 없고 드래그 핸들만 있다. 나머지 4개는 스위치 + 핸들.
- [ ] 재정렬한 순서가 draft 패널, 저장 draft 상세(`DraftDetailDialog`), 프리뷰, 8개 플랫폼 제출 본문에 동일하게 반영된다.
- [ ] 본문 구성 제목 우측 복원 버튼이 기본값일 때 비활성, 변경 시 활성이며 누르면 기본 순서로 되돌린다.
- [ ] 이슈 설정 탭에 "AI 설정" 섹션이 없고, "기타" 섹션이 최하단에 있으며 그 안에 `재현 과정 채우기` → `파일 첨부` 순으로 있다.
- [ ] 마이그레이션 후 기본 순서에서 8개 빌더 본문 출력이 변경 전과 바이트 동일하다(사전 박제 골든 통과).
- [ ] `pnpm test` 통과, `pnpm typecheck` 통과.
