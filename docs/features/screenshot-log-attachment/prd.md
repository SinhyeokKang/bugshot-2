# Screenshot Log Attachment

## 배경

현재 네트워크/콘솔 로그 첨부는 비디오 녹화 모드에서만 가능하다. 스크린샷 모드에서는 페이지의 네트워크 활동이나 콘솔 에러를 캡처할 방법이 없어, 개발자가 DevTools를 별도로 스크린샷해야 한다.

레코더 인프라(`network-recorder.ts`, `console-recorder.ts`)와 IndexedDB 저장(`blob-db.ts`), DraftingPanel 로그 UI(`LogAttachmentCards`, 프리뷰 다이얼로그)는 이미 비디오 모드용으로 구축되어 있다. 이 인프라를 스크린샷 모드로 확장한다.

## 목표

1. 사이드패널이 열려 있는 동안 네트워크/콘솔 활동을 지속 캡처한다 (백그라운드 레코딩).
2. 스크린샷 모드 DraftingPanel에서 축적된 로그를 이슈에 첨부할 수 있다.
3. 녹화 모드 진입 시 백그라운드 로그를 클리어하고 녹화 전용 로그로 교체한다.
4. URL 변경 시 로그를 초기화하고 레코더를 재주입한다.
5. 사이드패널 종료 시 pending 로그를 삭제한다.

## 비목표 (Non-goals)

- Element 모드 이슈에 로그 첨부 (추후 검토)
- 로그 시간 범위 필터링이나 선별 첨부
- Cross-tab 로그 공유 또는 origin 기반 로그 격리
- 백그라운드 서비스 워커에서의 독립적 로그 수집 (사이드패널 열림이 전제)
- 로그 첨부 기본값 on (기존 비디오 모드와 동일하게 기본 off, 사용자 토글)

## 사용자 시나리오

### 시나리오 1: 스크린샷 + 로그 첨부

1. 사용자가 사이드패널을 연다 → 네트워크/콘솔 백그라운드 캡처 시작
2. 웹 페이지 탐색 중 네트워크 에러 또는 콘솔 에러 발견
3. 스크린샷 모드 진입 → 영역 선택 → DraftingPanel 진입
4. DraftingPanel에 스크린샷과 함께 네트워크/콘솔 로그 카드 표시 (비디오 모드와 동일 UI)
5. 로그 카드 클릭 → 프리뷰 다이얼로그에서 내용 확인
6. 첨부 토글 활성화 → 이슈 등록 시 로그 포함

### 시나리오 2: 백그라운드 → 녹화 전환

1. 사이드패널 열림 → 백그라운드 레코딩 중
2. 녹화 모드 시작 → 백그라운드 로그 전부 클리어, 레코더 버퍼 초기화
3. 녹화 종료 → 녹화 구간 로그만 DraftingPanel에 표시
4. 녹화 이후 레코더 비활성. 다음 URL 이동 시 백그라운드 레코딩 재개

### 시나리오 3: URL 변경

1. 백그라운드 레코딩 중 사용자가 다른 URL로 이동 (SPA 라우팅 포함)
2. 기존 로그 삭제 (에디터 스토어 + IndexedDB pending)
3. 레코더 재주입 → 새 페이지에서 백그라운드 레코딩 재시작

### 시나리오 4: 사이드패널 종료

1. 사용자가 사이드패널을 닫음
2. IndexedDB의 `pending:{tabId}` 로그 삭제
3. 레코더 stop 신호 전송 (MAIN world에서 recording=false)
4. 재오픈 시 레코더 새로 주입, 빈 상태에서 축적 시작

### 시나리오 5: Element 모드 (변경 없음)

1. 백그라운드 레코딩 중 element 모드 진입
2. 로그는 백그라운드에서 계속 축적
3. Element 모드 DraftingPanel에는 로그 UI 미표시
4. Element 완료 후 스크린샷 모드 진입 시 축적된 로그 활용 가능
