# 캡처 모드 키보드 단축키

## 배경

현재 확장의 단축키는 `_execute_action`(`Cmd+Shift+E`) 하나뿐이고 사이드패널 토글에만 묶여 있다. 패널을 연 뒤 캡처를 시작하려면 매번 "이슈 작성" 진입 화면의 버튼 4개 중 하나를 마우스로 눌러야 한다. 자주 쓰는 3개 캡처 동작에 키보드 단축키를 붙이면 패널 토글 단축키와 한 세트로 키보드만으로 캡처를 시작할 수 있다.

## 목표

- 캡처 진입 동작 3개에 키보드 단축키를 부여한다. 캡처 진입 버튼 배열 순서(요소 → 화면 → 영상)대로 `1/2/3`을 배정한다.
  - 요소 선택 → `Cmd+Shift+1` (`Ctrl+Shift+1` on non-mac)
  - 화면 캡처 → `Cmd+Shift+2`
  - 영상 녹화 → `Cmd+Shift+3`
- 단축키는 해당 캡처 진입 버튼을 클릭한 것과 정확히 동일하게 동작한다.
- 단축키는 캡처 진입 화면이 보일 때만 발화한다 — 진행 중 작업을 덮어쓰지 않는다.
- 캡처 진입 버튼 3개에 마우스를 호버하면 현재 배정된 단축키를 툴팁으로 보여준다 — 사용자가 해당 캡처 모드를 처음 쓸 때 힌트로 파악할 수 있게 한다.

## 비목표 (Non-goals)

- **자유 작성(freeform)에는 단축키를 제공하지 않는다.** Chrome `suggested_key` 상한은 확장당 4개이며 `_execute_action` + 캡처 3개로 이미 가득 찬다. 자유 작성은 버튼으로만 유지하고, 단축키가 없으므로 툴팁도 붙이지 않는다.
- 단축키를 상시 노출(버튼 배지 등)하지 않는다 — 화면을 어지럽히지 않도록 **호버 툴팁으로만** 보여준다.
- 단축키 커스터마이즈 UI를 확장 안에 만들지 않는다. Chrome 기본 단축키 설정 화면(`chrome://extensions/shortcuts`)에 위임한다. 툴팁은 거기서 재바인딩한 키를 그대로 반영한다.

## 사용자 시나리오

### 주요 플로우

1. 사용자가 `Cmd+Shift+E`로 사이드패널을 연다 (또는 이미 열려 있다).
2. 패널이 `디버그` 메인탭의 `이슈 작성` 서브탭, 캡처 진입 화면(버튼 4개)을 보여준다.
3. `Cmd+Shift+1`을 누른다 → 요소 선택 picker가 시작된다 (버튼 클릭과 동일).
4. `Cmd+Shift+2` → 화면 캡처, `Cmd+Shift+3` → 영상 녹화도 동일하게 동작한다.

### 단축키 발견 플로우

1. 사용자가 캡처 진입 화면에서 "DOM 요소 선택" 버튼에 마우스를 올린다.
2. 툴팁이 떠서 현재 배정된 단축키(예: `⌘⇧1`)를 보여준다.
3. 이후 사용자는 마우스 없이 단축키로 캡처를 시작한다.
4. (`chrome://extensions/shortcuts`에서 키를 바꿨다면) 툴팁은 바뀐 키를 보여준다. 키를 해제했다면 툴팁은 뜨지 않는다.

### 엣지 케이스

- **패널이 닫혀 있음**: 단축키는 아무 동작도 하지 않는다 (no-op).
- **다른 메인탭 / 콘솔·네트워크 서브탭 조회 중**: 무시한다.
- **이미 요소 선택 / 드래프트 작성 중 등 진입 화면이 아님**: 무시한다 — 진행 중 작업 보호. 단, 요소 모드 진입 후 아직 선택 전 상태(`captureMode==="element" && !selection`)는 여전히 진입 화면으로 간주해 발화한다.
- **여러 창에 패널이 떠 있음**: 단축키를 누른 창의 패널만 반응한다.
- **지원하지 않는 페이지(chrome:// 등)**: 패널이 미지원 화면을 보이며, 단축키 핸들러가 마운트되지 않아 무시된다.
- **단축키 미배정**: `chrome://extensions/shortcuts`에서 키 충돌·해제로 커맨드에 키가 없으면 `onCommand`가 발화하지 않아 자연 no-op이며, 해당 캡처 버튼의 툴팁도 표시되지 않는다.

## 성공 기준

- `chrome://extensions/shortcuts`에 캡처 커맨드 3개가 `Cmd+Shift+1/2/3`으로 자동 배정되어 노출된다 (충돌 없음).
- 캡처 진입 화면에서 각 단축키가 대응 버튼과 동일하게 동작한다.
- 캡처 진입 버튼 3개 호버 시 현재 단축키가 툴팁으로 표시된다.
- 진입 화면이 아닌 상태·다른 탭·패널 닫힘에서 단축키가 무시되고 콘솔 에러가 발생하지 않는다.
- 키가 미배정된 커맨드는 호버 툴팁이 표시되지 않는다 (`useCommandShortcuts`가 빈 `shortcut`을 제외).
- 영상 단축키(`⌘⇧3`)가 `tabCapture` 제약으로 실패하면 버튼 클릭과 동일하게 `onVideoRecordingUnavailable` 다이얼로그로 graceful하게 처리된다.
- `resolveCaptureShortcut` 순수 함수 단위 테스트 통과, `pnpm typecheck` 클린.
