# Slack 연동

## 배경

Bugshot은 현재 6개 이슈 트래커(Jira·GitHub·Linear·Notion·GitLab·Asana)로 이슈를 등록할 수 있다. 그러나 많은 팀이 버그 리포트를 **이슈 트래커가 아니라 Slack 채널/DM으로 먼저 공유**한다 — 가벼운 공유, 담당자 태깅, 빠른 논의가 메시지 앱에서 일어난다.

문제: Slack은 이슈 트래커가 아니라 **메시지 앱**이다. 이슈 객체·상태(open/done)·담당자 같은 구조가 없다. "타 플랫폼과 동일 수준으로 보낼 수 있느냐"는 곧 *우리가 조립한 이슈 본문(제목·환경·재현 절차·스크린샷·영상·로그)을 Slack 메시지로 손실 없이 옮길 수 있느냐*의 문제다.

## 목표

- 기존 6개 플랫폼과 **동일한 진입점**(이슈 작성 → 플랫폼 선택 → 전송)으로 Slack 채널에 이슈를 전송할 수 있다.
- 이슈 본문을 **Block Kit으로 구조화**해 제목은 header 블록, 환경 정보·섹션·스타일 diff는 각각 별도 section 블록으로 분리되어 표시된다.
- 스크린샷·before/after·30s Replay 영상·로그 파일을 **본문 메시지의 스레드에 첨부**한다.
- 전송 후 이슈 리스트에 **"전송됨"으로 기록**하고, 메시지 permalink를 URL로 저장해 클릭 시 Slack으로 이동한다. (Slack 항목은 추적 대상 이슈가 아닌 **전송 기록**으로 동일 리스트에 표시 — 상태 폴링·토글 없음.)
- 사용자 **본인 명의(user token)**로 전송해, 본인이 속한 채널에 봇 초대 없이 바로 보낸다.

## 비목표 (Non-goals)

- **상태 추적·동기화 없음**: Slack 메시지는 open/done 같은 상태가 없다. 이슈 리스트에서 슬랙 항목은 상태 폴링·토글을 하지 않는다 (정적 "전송됨" 배지 + permalink만).
- **DM 전송 미지원 (1차)**: 이번 스코프는 **채널(public/private)만**. DM(`im`)·그룹DM(`mpim`)은 상대 user ID만 줘 `users.info` resolve가 필요하고 대형 워크스페이스에서 N+1 비용이 커 후속으로 미룬다.
- **봇 명의 전송 미지원**: 봇 토큰(xoxb)·봇 채널 초대 플로우는 이번 스코프에서 제외. user token만.
- **Slack self-managed/Enterprise Grid 특수 처리 없음**: 표준 `slack.com` 워크스페이스만.
- **메시지 인라인 이미지 미지원**: Slack Block Kit `image` 블록은 외부 공개 URL만 받는다. `files` API로 올린 파일은 비공개 URL이라 본문 인라인 불가 → 전부 스레드 첨부로 처리한다.
- **메시지 수정/삭제·답글 추적 없음**: 전송은 1회성. 이후 메시지 편집·재전송 동기화 안 함. **재전송 시 새 메시지가 또 생성된다**(중복 허용, 기존 6개 플랫폼과 동일하게 재전송 가능).
- **슬래시 커맨드·인터랙티브 컴포넌트 없음**: 버튼·모달 등 Slack 앱 인터랙션 미사용.

## 사용자 시나리오

### 주 플로우: Slack으로 이슈 전송

1. 사용자가 Integrations 탭 → "플랫폼 추가"에서 Slack을 선택, OAuth로 본인 워크스페이스 계정을 연결한다. (연결됨 화면에서 기본 전송 채널을 미리 지정해둘 수 있다.)
2. DOM 요소를 고르고 스타일을 수정하거나 스크린샷/영상을 캡처해 이슈 본문(제목·섹션)을 작성한다.
3. "Submit" → 플랫폼 선택 다이얼로그에서 **Slack 탭**을 고른다.
4. 전송 대상 채널을 콤보박스에서 검색·선택한다 (기본 채널이 있으면 prefill).
5. 전송 → Block Kit 본문 메시지가 채널에 게시되고, 스크린샷·영상·로그가 그 메시지의 **스레드 답글로 첨부**된다.
6. 완료 화면에서 메시지 링크를 확인한다. 이슈 리스트에 "전송됨"으로 기록된다.

### 엣지 케이스

- **첨부 일부 실패**: 파일 업로드가 개별 실패해도 본문 메시지는 보존된다 (per-file 격리, Asana 패턴).
- **영상 용량 초과**: Slack 파일 한도 초과 시 Slack API 에러를 그대로 안내. 본문 메시지·다른 첨부는 보존.
- **대상 미선택**: 채널을 고르지 않으면 전송 버튼 비활성 (`canSubmit` 가드).
- **토큰 만료/철회**: 사용자가 Slack에서 앱 권한을 철회하면 API가 `invalid_auth`/`token_revoked` 반환 → 재연결 안내. (자동 refresh 없음 — user token은 기본 무기한이므로 만료는 곧 철회.)
- **재전송**: 이슈 리스트/드래프트에서 Slack 항목을 재전송하면 **새 메시지가 다시 게시**된다(중복 허용). 기존 메시지 편집은 하지 않는다.

## 성공 기준

- Slack을 연결한 뒤 채널(public/private)에 이슈 본문 메시지 + 스레드 첨부를 전송할 수 있다.
- Block Kit 본문에 제목(header 블록)·환경 정보·섹션·스타일 diff(각 section 블록)가 분리·구조화되어 보인다.
- 스크린샷·영상·로그가 스레드에 첨부되고 Slack 미리보기가 동작한다.
- 전송 후 이슈 리스트에 슬랙 항목이 "전송됨" + 클릭 가능한 permalink로 남는다.
- 기존 6개 플랫폼 전송 플로우에 회귀가 없다 (`PlatformId` union 확장으로 인한 누락 분기 없음).
- `markdownToSlackBlocks` 순수 함수 단위 테스트가 통과한다.
