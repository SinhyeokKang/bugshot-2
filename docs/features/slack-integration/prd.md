# Slack 연동

## 배경

Bugshot은 현재 7개 이슈 트래커(Jira·GitHub·Linear·Notion·GitLab·Asana·ClickUp)로 버그를 등록한다. 하지만 많은 팀은 버그를 트래커에 정식 등록하는 대신(혹은 그에 앞서) **Slack 채널에 가볍게 공유**한다 — "이거 깨졌어요 @담당자" 한 줄 + 스크린샷. 트래커 연동만으로는 이 가벼운 공유 흐름을 못 잡는다. Slack을 8번째 전송 대상으로 추가하면, 사용자는 캡처한 버그 컨텍스트(환경·스타일 diff·로그·스크린샷/영상)를 곧장 Slack 채널/DM에 던지고 담당자를 멘션으로 호명할 수 있다.

경쟁 도구(예: Traceo)는 트래커 연동이 아예 없고, 이슈 트래커 도구는 메신저 공유가 없다. Bugshot은 **트래커 등록과 Slack 공유를 한 도구에서** 제공해 둘 사이 갭을 메운다 — 사용자는 캡처 단위로 둘 중 어디로 보낼지 고른다.

Slack은 이슈 트래커가 아니라 메시지 앱이라, 기존 7개의 "이슈 생성" 모델과 구조가 갈린다. 라벨·담당자·상태 같은 트래커 개념이 없고, 대신 **채널 / 스레드 / 메시지 / 멘션** 모델을 쓴다. 본 기능은 기존 어댑터 패턴을 최대한 재사용하되, 메시지 앱 특성에 맞는 지점만 다르게 구현한다.

### Slack ↔ 트래커 관계 (단일 제출 모델)
Slack은 다른 7개 플랫폼과 **동등한 택1 최종 전송처**다. 한 캡처는 한 곳에만 전송된다. 제출 시 기존 동작대로 로컬 캡처 데이터(스크린샷·영상·로그·스타일 diff blob)가 IndexedDB에서 삭제되므로(`stripSubmitted`), **제출 후 다른 플랫폼으로의 재제출/승격은 지원하지 않는다** — 기존 BugShot도 제출된 이슈엔 재제출 UI가 없고(`IssueRow`는 submitted 이슈 클릭 시 URL만 연다), 원본 blob이 사라져 구조적으로 불가능하다. 이슈 행 배지는 제출 플랫폼 1개만 표시 — Slack 전용 분기·이중 배지는 추가하지 않는다.

> '1차 공유 후 트래커 승격' 같은 다중 제출 워크플로우는 데이터 보존 모델(`stripSubmitted`/`IssueRecord`) 변경이 필요해 **이번 스코프 밖**이다(필요 시 별도 feature로 분리).

## 목표

- 사용자가 OAuth로 본인 Slack 계정을 연결하면, **본인 이름으로**(user token) 메시지를 보낼 수 있다.
- 캡처한 버그를 Slack의 **public 채널 / private 채널 / DM**에 전송한다.
- 전송 구조: **제목(+멘션)은 부모 메시지, 상세 본문(환경·스타일 diff·로그 요약)과 첨부(스크린샷·영상·logs.html)는 같은 메시지의 스레드 답글**로 보낸다 → 채널 타임라인은 제목 한 줄로 깔끔하고, 상세는 스레드에 접힌다. 멘션은 부모 메시지에 넣어 호명된 사람에게 알림이 확실히 가게 한다.
- 선택한 워크스페이스 멤버를 **멘션 대상**으로 지정하면 부모 메시지에 `<@Uxxxx>`로 호명된다(알림 발송).
- 기존 마크다운 본문을 Slack **mrkdwn** 포맷으로 정확히 변환한다.
- 전송 후 생성된 메시지의 **permalink**를 이슈 행에 노출해 클릭 시 Slack으로 이동한다.
- 기존 7개 플랫폼의 어댑터·OAuth·UI·i18n·store 패턴을 최대한 그대로 따른다(신규 추상화 도입 금지).

## 비목표 (Non-goals)

- **Bot token 전송 모델** — bot이 채널에 초대돼야 하는 마찰이 있어 1차 제외. user token만.
- **BYOK(토큰 직접 입력)** — Slack user token(xoxp)은 사용자가 직접 앱을 만들어야 발급 가능해 진입장벽이 높다. OAuth 전용. (다른 플랫폼의 PAT 분기는 추가하지 않는다.)
- **담당자·라벨·우선순위·상태** — Slack 메시지 모델에 없는 트래커 개념. 필드 추가하지 않는다. (멘션은 별도 필드로 지원 — 비목표 아님.)
- **멤버 외 채널 탐색/전송** — `users.conversations` 기반이라 **본인이 멤버인 대화만** 목록에 뜨고 전송 가능. 워크스페이스 전체 채널 검색·미가입 채널 전송은 미지원.
- **메시지 상태 추적/폴링** — Slack 메시지엔 open/closed 같은 상태가 없다. 전송됨 배지 + permalink 링크만. (기존 배지의 상태 폴링 로직을 Slack엔 적용하지 않는다 — `isRefreshable`이 slack을 false로 폴백하는 게 **의도된** 동작.)
- **본문 inline 이미지 인라인 렌더** — mrkdwn은 인라인 이미지를 지원하지 않는다. 모든 이미지는 스레드 첨부로 보낸다(ClickUp의 2차 본문 갱신 패턴 미적용).
- **메시지 편집/삭제, 스레드 추가 답글, 리액션** — 전송 1회로 끝.
- **여러 워크스페이스 동시 연결** — OAuth로 인증한 단일 워크스페이스만.

## 사용자 시나리오

### 1. Slack 연결 (OAuth)
1. 사용자가 Integrations 탭에서 "Slack 연결"을 누른다.
2. `chrome.identity.launchWebAuthFlow`로 Slack OAuth v2 동의 화면이 열린다 (user_scope: `chat:write, channels:read, groups:read, im:read, mpim:read, files:write, users:read`).
3. 동의하면 user token(`xoxp-...`)과 워크스페이스 정보(team id/name), 본인 정보(user id/name)를 받아 저장한다.
4. Integrations 탭에 연결된 워크스페이스 이름과 기본 채널 선택 UI가 보인다.

### 2. 버그를 Slack에 전송
1. 캡처(element/screenshot/video/freeform) 후 이슈 작성 모달에서 플랫폼 탭 "Slack"을 고른다.
2. 채널 선택 콤보박스에서 전송 대상(공개/비공개 채널 또는 DM)을 고른다. 본인이 멤버인 대화만 목록에 뜬다(플랫 리스트 + 종류 아이콘).
3. (선택) 멘션 대상 멤버를 고른다 — 부모 메시지에 `<@Uxxxx>`로 호명된다.
4. 제목과 본문을 작성하고 전송한다. (제목은 필수 — 부모 메시지가 되므로 비면 전송 비활성.)
5. 결과:
   - 채널에 **제목(+멘션)**이 부모 메시지로 올라간다.
   - 그 메시지의 **스레드**에 본문(환경 정보 + 사용자 작성 섹션 + 스타일 diff 텍스트 + 로그 요약)이 mrkdwn으로 올라간다.
   - 스레드에 스크린샷(before/after), 영상, logs.html이 파일로 첨부된다.
6. 이슈 행에 Slack 칩 + "전송됨" 배지가 뜨고, 클릭하면 메시지 permalink로 이동한다.

### 엣지 케이스
- **채널 목록이 많음**: `users.conversations`는 커서 페이지네이션 → 전부 모아 콤보박스에 표시.
- **DM 이름**: im(1:1)은 상대 user 이름, mpim(그룹 DM)은 참여자 라벨로 표시. 이름 해석은 `users.list`를 1회 받아 user id→name으로 매핑(N+1 회피)하고 결과를 캐시. 채널은 먼저 표시하고 이름은 배치로 채운다.
- **빈 제목**: 제목은 부모 메시지라 빈 값이면 전송 비활성(필수 검증).
- **멘션 대상 없음**: 멘션은 선택 — 미지정 시 부모 메시지는 제목만.
- **토큰 revoke**: user가 Slack에서 앱 권한을 철회하면 API가 `ok:false, error:"token_revoked"` → 재연결 안내 에러(ClickUp `oauthRevoked` 패턴).
- **파일 업로드 실패**: 부모/본문 메시지는 보존하고, logs.html이 누락되면 `logsDropped: true`로 사용자에게 알린다(기존 패턴).
- **스레드 답글 본문이 비어도** 환경 정보는 항상 있으므로 빈 스레드는 발생하지 않는다.
- **채널 미선택 전송 시도**: 필수 검증으로 차단(`create.requiredMissing`).

## 성공 기준

- OAuth로 Slack 워크스페이스를 연결하고 user token을 저장할 수 있다.
- public/private 채널과 DM 목록을 콤보박스에서 선택할 수 있다.
- 전송하면 채널에 제목(+멘션) 부모 메시지 + 스레드에 본문/첨부가 정확히 올라간다.
- 멘션 대상으로 고른 멤버가 부모 메시지에서 `<@Uxxxx>`로 호명돼 알림을 받는다.
- 본문 mrkdwn 변환이 규칙표의 각 케이스(볼드/이탤릭/취소선/링크/헤딩→볼드/리스트/코드블록 보존/이미지 제거/`<>&` 이스케이프)에 대해 **단위 테스트로 green**이다.
- 이슈 행의 Slack 배지를 클릭하면 메시지 permalink로 이동한다.
- 기존 7개 플랫폼 전송이 회귀 없이 동작한다(`PlatformId` union 확장으로 인한 exhaustive switch 누락 없음).
- `pnpm typecheck`·`pnpm test` 통과.
