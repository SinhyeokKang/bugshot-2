# AI 초안 생성 — 선입력 컨텍스트 활용 & 이미지 보존

## 배경

drafting에서 사용자가 제목·본문을 일부 작성하고 이미지까지 첨부한 뒤 "AI 초안 생성"을 누르면 현재는:

1. **선입력 값이 LLM에 전달되지 않는다.** `buildAiDraftSessionPrompt`는 페이지/캡처/로그만 컨텍스트로 넘기고 사용자가 적어둔 `draft.title`·`draft.sections`는 무시한다. AI가 사용자가 이미 정리한 맥락을 못 본다.
2. **응답이 모든 본문을 통째로 덮어쓴다.** `parseAiDraftResponse` 결과로 `draft.sections`를 완전 교체해서, 사용자가 본문에 삽입한 inline 이미지(`![](inline:ref)`)가 사라진다.
3. **이미지가 있던 섹션은 화면 반영조차 안 된다.** `TiptapEditor`의 `isInternalChange` 플래그가 inline 이미지 resolve 후 `true`로 잔류해, 직후 외부 `value` 변경(AI 덮어쓰기)을 1회 무시한다. store는 바뀌어도 에디터 표시가 안 바뀌어 "응답 누락"처럼 보인다.

## 목표

- AI 초안 생성 요청 시 사용자가 선입력한 **텍스트(title + 각 섹션)와 이미지**를 LLM 컨텍스트로 함께 전달한다.
- LLM 응답은 **텍스트만** 각 섹션에 덮어쓴다. 기존 이미지는 보존한다.
- 보존된 이미지는 해당 섹션 본문 **맨 위**에 두고, 그 아래에 LLM 응답 텍스트가 온다.
- 이미지가 있던 섹션도 AI 응답이 에디터에 정상 반영된다(`TiptapEditor` 잔류 플래그 버그 수정).

비주얼:
```
[선입력 텍스트]          →   LLM 요청(텍스트+이미지 컨텍스트)   →   [선 업로드 이미지]
[선 업로드 이미지]                                                  [LLM 응답 텍스트]
```

## 비목표 (Non-goals)

- 적용 전 확인 다이얼로그·미리보기·Undo 도입 (덮어쓰기는 유지).
- LLM 응답이 이미지를 생성/재배치하도록 하는 것 (응답은 텍스트만).
- orderedList 섹션(`stepsToReproduce`)의 이미지 처리 — 해당 섹션은 이미지를 가질 수 없다.
- AI 스타일링(`AiStylingDialog`) 경로 변경 — 이번 스코프는 `AiDraftDialog`만.
- 이미지 전송 개수/용량을 사용자가 설정하는 옵션.

## 사용자 시나리오

### 주 시나리오: 텍스트+이미지 선입력 후 AI 생성
1. 사용자가 description 섹션에 텍스트를 적고 이미지 1장을 첨부한다.
2. "AI 초안 생성" → 다이얼로그에 추가 설명 입력 후 생성.
3. LLM에 선입력 텍스트(이미지 ref 제거됨) + 선업로드 이미지 + 캡처 이미지가 전달된다.
4. 응답 후 description = `[선업로드 이미지]` + 빈 줄 + `[LLM 텍스트]`. 이미지는 그대로, 텍스트만 교체.

### 엣지 케이스
- **이미지 없는 섹션**: 기존대로 LLM 텍스트로 전체 교체.
- **이미지만 있고 텍스트 없던 섹션**: 이미지 보존 + 그 아래 LLM 텍스트 추가.
- **LLM이 해당 섹션 응답을 누락**: 텍스트는 빈 문자열로 취급하되 이미지는 보존(이미지만 남음).
- **선입력이 전혀 없음**: 컨텍스트 블록 생략, 기존 동작과 동일.
- **여러 이미지**: 원본 본문 등장 순서대로 상단에 나열.
- **orderedList 섹션**: 이미지 불가 → 기존대로 텍스트 교체.

## 성공 기준

- 선입력 텍스트·이미지가 LLM 요청에 포함된다(요청 payload로 검증 가능).
- 이미지가 있던 paragraph 섹션은 AI 생성 후에도 이미지가 유지되고, 이미지 아래에 새 텍스트가 위치한다.
- 이미지가 있던 섹션이 AI 생성 직후 에디터 화면에 즉시 반영된다.
- 이미지 없는 섹션은 기존과 동일하게 전체 교체된다.
- 신규 순수 함수 단위 테스트가 모두 통과하고 `pnpm test` green.
