# 슬랙 승격 백링크 (Slack Promotion Thread Link)

## 배경

Slack으로 공유한 이슈는 보존(`slackPreserved`)되어 나중에 Jira·GitHub 등 트래커로 "승격"할 수 있다. 하지만 승격해도 **원 Slack 메시지에는 아무 흔적이 남지 않는다**. 슬랙에서 버그를 보고 토론하던 팀원은 그 버그가 트래커에 등록됐는지, 어디에 등록됐는지 알 수 없다. 슬랙 스레드와 트래커 이슈가 단절돼 추적이 끊긴다.

## 목표

- 슬랙 보존 이슈를 트래커로 승격하면, **원 슬랙 메시지 스레드에 등록된 트래커 이슈 링크를 댓글로 남긴다**.
- 슬랙 ↔ 트래커 양방향 추적: 슬랙에서 원 메시지만 봐도 "이건 처리됐고 여기 있다"를 알 수 있다.
- 댓글 작성은 **best-effort**다. 실패해도 승격 자체는 정상 성공으로 처리하고 사용자에게 알리지 않는다.

## 비목표 (Non-goals)

- 일반(비-Slack) 이슈 제출에는 적용하지 않는다. 슬랙 보존 이슈의 승격에만.
- 슬랙 외 다른 플랫폼으로의 역링크(예: Jira 코멘트에 슬랙 링크)는 이번 스코프 밖.
- 댓글 작성 실패에 대한 재시도·큐잉·사용자 알림·로깅 UI 없음. 조용히 drop.
- 댓글 문구 커스터마이즈 설정 없음.
- 원 메시지가 삭제됐는지 사전 확인하지 않는다(확인 호출 추가 안 함 — 실패하면 그냥 drop).

## 사용자 시나리오

**주 플로우:**
1. 사용자가 이슈를 Slack 채널에 공유한다 → 이슈가 `slackPreserved`로 목록에 남는다.
2. 이슈 카드의 [승격] 버튼 → 제출 다이얼로그에서 트래커(예: Jira) 선택 후 제출.
3. 트래커 등록 성공 → 백그라운드에서 원 슬랙 메시지 스레드에 댓글이 달린다:
   ```
   Jira에 이슈로 등록되었습니다.
   https://your.atlassian.net/browse/BUG-123
   ```
4. 사용자는 평소처럼 성공 화면을 본다(댓글 동작은 비노출).

**엣지 케이스 (모두 조용히 drop — 승격은 성공):**
- 슬랙 OAuth 토큰이 없거나(연결 해제됨) 만료/폐기됨.
- 원 메시지 permalink(`issue.url`)에서 channel을 파싱하지 못함.
- 원 메시지가 삭제되어 `chat.postMessage`가 실패함.
- 봇/유저가 해당 채널 멤버가 아님(`not_in_channel`).
- 네트워크 오류·rate limit 등 임의의 Slack API 실패.

## 성공 기준

- 슬랙 보존 이슈를 트래커로 승격하면, 원 메시지 스레드에 트래커 URL을 포함한 댓글이 1개 달린다.
- 위 엣지 케이스 어디서든 예외가 사용자 플로우로 전파되지 않고, 승격 결과(성공 화면)는 동일하게 표시된다.
- 일반 이슈 제출·Slack 공유 자체에는 동작 변화가 없다.
