# 스타일 변경사항 확인 다이얼로그

## 배경

element mode의 styling 단계에서 사용자는 여러 요소를 옮겨가며 스타일을 수정한다. 현재 변경 내역을 한눈에 볼 방법이 없다 — PageFooter의 [변경사항 초기화] 버튼은 건수만 보여주는 AlertDialog를 거쳐 전부 되돌리는 all-or-nothing 동작뿐이다. 무엇을 바꿨는지 확인하고 일부만 되돌리려면 각 요소를 다시 선택해 패널을 뒤져야 한다.

## 목표

- PageFooter의 [변경사항 초기화] 버튼을 **[변경사항 확인 · N]** 버튼으로 교체한다. N은 회색 배경 badge로 표시되는 변경 항목 수(스타일 diff 행 수)다.
- 변경이 0건이면 badge를 표시하지 않고 버튼을 비활성화한다.
- 버튼 클릭 시 Dialog에서 **요소 단위 그룹** 아래 **스타일 항목(diff 행) 단위**로 변경 내역을 리스팅한다. 현재 선택 요소와 버퍼된 요소 모두 포함한다.
- 각 diff 행은 muted 컨테이너 우측 [x]로 개별 초기화할 수 있다. 초기화하면 페이지 DOM이 즉시 원복되고 스타일 패널 인풋도 갱신된다.
- 각 요소 카드 우상단 [x]로 해당 요소의 모든 스타일 변경을 일괄 초기화할 수 있다.
- 행 [x]·카드 [x] 모두 별도 확인(AlertDialog) 없이 즉시 실행된다. 재확인은 [전체 초기화]에만 있다.
- 버퍼된 요소의 변경 항목을 개별 초기화하면 해당 요소의 afterImage를 **재캡처해 갱신**한다(재캡처 실패 시 기존 이미지 유지). 요소의 변경이 전부 사라지면 버퍼에서 항목을 제거한다(이미지 포함 폐기).
- 다이얼로그 하단: 가장 좌측에 destructive [전체 초기화](AlertDialog 재확인 후 모든 변경 원복), 가장 우측에 [확인](다이얼로그 닫기).
- 전체/개별 초기화로 변경이 0건이 되면 다이얼로그는 자동으로 닫힌다.
- 전체 초기화 후에도 현재 선택(selection)은 유지된다 — picking 상태로 돌아가지 않는다.

## 비목표 (Non-goals)

- 다이얼로그 내 before/after 스냅샷 이미지 표시 (diff 행만 보여준다).
- 버퍼된 요소를 다이얼로그에서 재선택/재편집으로 진입하는 기능.
- drafting 이후 단계(DraftingPanel·PreviewPanel)의 변경 내역 UI 변경.
- 값 수정(편집)을 다이얼로그 안에서 하는 기능 — 초기화만 가능.

## 사용자 시나리오

### 기본 흐름
1. styling 단계에서 요소 A의 color·padding을 수정하고, 요소 B로 전환해 class를 수정한다 (A는 버퍼로 들어감).
2. PageFooter에 [변경사항 확인 · 3] 버튼이 보인다 (A: color, padding / B: class).
3. 클릭하면 Dialog가 열리고 요소별 그룹(A, B) 아래 각 변경 행이 as-is → to-be로 나열된다.
4. A의 padding 행 우측 [x]를 누르면(재확인 없음): 페이지에서 A의 padding이 원래 값으로 돌아가고, A의 afterImage가 재캡처되며, badge가 2로 줄어든다.
5. B 카드 우상단 [x]를 누르면(재확인 없음): B의 모든 변경이 원복되고 B 카드가 사라지며 badge가 1로 줄어든다.
6. [확인]을 누르면 다이얼로그가 닫히고 나머지 변경은 유지된다.

### 전체 초기화
1. 다이얼로그 좌하단 [전체 초기화](destructive)를 누른다.
2. AlertDialog로 "N건의 변경사항을 초기화하시겠습니까?" 재확인.
3. 확인하면 모든 요소의 DOM이 원복되고 버퍼가 비워지며, 다이얼로그가 자동으로 닫힌다. 현재 선택 요소는 그대로 선택되어 있고 스타일 패널 인풋은 원래 값으로 갱신된다.

### 엣지 케이스
- **마지막 항목 개별 초기화**: 변경이 0건이 되면 다이얼로그 자동 닫힘 + footer 버튼 비활성화(badge 미노출).
- **버퍼 요소가 뷰포트 밖**: afterImage 재캡처 전 해당 요소로 스크롤 → 캡처 → 원래 스크롤 위치 복원.
- **버퍼 요소가 DOM에서 사라짐**(페이지 변형) 또는 **재캡처 실패**(captureVisibleTab quota 등): DOM 원복·재캡처는 스킵하고 store의 변경 항목만 갱신, afterImage는 **기존 이미지를 그대로 유지**한다(실제 화면과 다를 수 있음을 수용).
- **버퍼된 요소를 재선택해 편집 중**(같은 요소가 버퍼 항목 + 현재 선택으로 중복): 버퍼 항목의 과거 편집도 카드로 표시하고 행 [x]·카드 [x] 초기화를 허용한다. 초기화 시 요소를 강제 재선택(re-emit)해 selection·스타일 패널 인풋을 새 베이스라인으로 갱신한다. **이때 현재 작성 중이던(아직 버퍼에 안 들어간) 편집은 폐기된다.**
- **카드 [x]로 요소 초기화**: 마지막 행 [x]와 동일한 결과 — 버퍼 요소면 버퍼에서 제거(이미지 폐기), 현재 선택 요소면 styleEdits만 베이스라인으로 원복하고 선택은 유지.
- **요소의 text가 없는 경우**(selection.text === null): text 행은 애초에 diff에 없으므로 영향 없음.

## 성공 기준

- [변경사항 확인 · N]의 N = 다이얼로그에 나열되는 행 수와 항상 일치한다 (shorthand collapse 반영).
- 개별 초기화 직후: 페이지 DOM·스타일 패널 인풋·badge 카운트·버퍼 데이터가 모두 일관된 상태가 된다.
- 버퍼 요소 개별 초기화 후 afterImage가 현재 화면과 일치한다 (재캡처 성공 시).
- 전체 초기화는 기존 [변경사항 초기화]와 동일한 결과를 낸다 (selection 유지 + DOM 전체 원복 + 버퍼 비움).
- `pnpm test` 통과 — 신규 순수 헬퍼(그룹 빌드·카운트·행 제거)의 단위 테스트 포함.
- `pnpm typecheck` 통과.
