# PRD — 비디오 모드 HTML 리포트

## 배경

비디오 모드로 캡처한 버그 이슈를, 받는 사람이 BugShot 설치 없이 브라우저에서 바로 열어볼 수 있는 **단일 HTML 파일**로 내보낸다. HTML 안에서 녹화 영상을 재생하고, 네트워크·콘솔 로그를 영상 타임라인과 양방향 동기화해 "언제 무슨 일이 일어났는지"를 직관적으로 파악하게 한다.

비디오 모드는 이미 출시돼 있지만 공유 경로가 약하다 — 영상·로그가 영상 타임라인과 묶인 채로 나가는 길이 없다. 기존 공유 수단의 한계:

- **마크다운 복사**: 영상–로그 시간 동기화가 사라진다.
- **플랫폼 연동(Jira/GitHub/Linear/Notion)**: 계정 없는 외부 클라이언트한테는 못 보낸다.
- **백엔드 호스팅 옵션**: 인프라·DB·인증 부담.

→ 단일 HTML이 가장 가볍게 인터랙션을 유지하는 길이며, "브라우저를 떠나지 마세요"라는 BugShot 콘셉트와도 일관된다.

## 목표

비디오 모드 이슈 1건을 의존성 없는 단일 HTML 파일로 만들고, 그 파일을 받은 사람이 어떤 브라우저에서든 더블클릭만으로:

1. 버그 녹화 영상을 재생할 수 있다.
2. 네트워크·콘솔 로그가 영상 시간축과 동기화되어, 특정 로그가 영상의 몇 초 시점에 발생했는지 즉시 파악할 수 있다.

리포트는 두 경로로 산출된다 — **Preview 패널에서 직접 다운로드**, 그리고 **이슈 제출 시 플랫폼 이슈에 자동 첨부**.

## 사용자 시나리오

### 시나리오 A — 제출 전 직접 공유

QA가 비디오 모드로 버그를 녹화하고, 플랫폼에 올리기 전 Preview 패널에서 HTML 리포트를 다운로드해 개발자·디자이너·외부 협력사에게 파일로 직접 전달한다. 받는 사람은 확장 설치·로그인·계정 없이 파일을 열어, 영상을 보며 "이 에러 로그가 클릭 직후에 떴구나"를 눈으로 확인한다.

### 시나리오 B — 제출 시 자동 첨부

QA가 비디오 모드 이슈를 Jira/GitHub/Linear/Notion에 제출하면, HTML 리포트가 `bugshot-report.html`로 그 이슈에 자동 첨부된다. 제출 후 BugShot은 영상·로그 blob을 로컬에서 정리하지만, 리포트가 플랫폼 이슈에 남아 있어 팀원이 나중에 내려받아 더블클릭으로 영상–로그 동기화 화면을 다시 볼 수 있다.

## 스코프

### In (V1)

- **비디오 모드 이슈 한정** — element/screenshot 모드는 대상 아님.
- 단일 HTML 파일 — 외부 의존성 0 (인라인 CSS/JS, 영상은 dataUrl 인라인).
- 영상 ↔ 로그 양방향 시간 동기화.
- 타입별 로그 필터 — 기존 `ConsoleLogContent`/`NetworkLogContent`의 분류 체계와 정렬.
- 키보드 단축키 (재생·점프).
- 진입점 2곳:
  - **Preview 패널** — 제출 전 draft를 HTML 파일로 로컬 다운로드.
  - **제출 시 자동 첨부** — 비디오 모드 이슈를 제출하면 4개 플랫폼 모두에 HTML 리포트를 파일 첨부.
- 타임라인·로그 목록은 **녹화 구간 내 로그만** 표시.
- 로그 0건·필터 결과 0건·영상 로드 실패에 대한 빈 상태/graceful degradation.

### Out (이후 버전 검토)

- 제출된 이슈를 IssueListTab에서 다시 export — 제출 시 영상·로그 blob이 정리되므로 사후 재가공 불가. 자동 첨부가 이 필요를 대체.
- CSS diff 테이블·스크린샷 갤러리 (element 모드 데이터 — 비디오 모드 이슈엔 없음).
- zip 폴백 (용량 초과 시 분리 패키징).
- 영상 트리밍·편집.
- 코멘트·답글.
- 인증·접근 권한 (정적 파일이라 누구나 봄).
- 분석·추적.
- 리포트 UI 다국어 (영문 단일로 시작).
- 리포트 재import·재가공.

## 성공 기준

- Preview 패널에서 비디오 이슈를 HTML 파일로 다운로드할 수 있다.
- 비디오 모드 이슈를 제출하면 4개 플랫폼 이슈에 `bugshot-report.html`이 첨부된다.
- 내보낸 HTML을 Chrome·Firefox·Safari에서 열어 영상이 재생되고 로그 동기화가 동작한다.
- 로그 행 클릭 시 영상이 해당 시각으로 점프하고, 영상 재생 시 현재 시각의 로그 행이 하이라이트된다.
- 녹화 구간 밖 로그는 리포트에 포함되지 않는다.
- 로그를 첨부하지 않은 비디오 이슈도 영상만 담긴 리포트로 정상 export된다 (graceful degradation).
- 로그 0건·필터 결과 0건일 때 우측 패널이 빈 상태를 명시적으로 표시한다.
- 영상 로드에 실패해도 로그 패널·타임스탬프 목록은 계속 보인다.

## 열린 질문 — 결론

| 질문 | 결론 |
|---|---|
| 녹화 코덱이 받는 사람 환경에서 안전한가 | 녹화는 MP4/H.264 우선이라 대부분 안전. WebM로 떨어진 경우만 Safari 재생 위험 → 리포트가 `video` 에러 시 코덱 안내 노출 + 로그 패널은 살려둠. |
| dataUrl 영상이 보안상 차단되는가 | 로컬 `file://` HTML 더블클릭 시나리오엔 CSP 미적용 → 재생됨. |
| 리포트 용량이 25MB를 초과하면 | 다운로드는 그대로 진행하되, 확장 UI에 "메일 첨부가 제한될 수 있다"는 토스트 경고 노출. zip 분리는 V1 제외. |
| 제출 시 첨부한 리포트가 플랫폼 첨부 용량 한도를 넘으면 | **위험으로 식별** — Jira Cloud 기본 첨부 한도(10MB)·GitHub 등 플랫폼별 한도가 영상 크기보다 작을 수 있다. V1은 첨부를 시도하고 실패 시 이슈 본문/AI 메타 첨부는 정상 진행(리포트 첨부만 누락). 한도 사전 차단·압축은 후속 검토. design.md 위험 요소 참조. |
| 자동 스크롤 기본값 | 기본 ON, 사용자가 수동 스크롤하면 OFF + "Resume" 플로팅 버튼으로 복귀. |
| 타임라인 마커 클릭 동작 | 마커는 호버 미리보기만, 영상 점프는 로그 행 클릭으로 받는다 (마커 겹침 시 정밀 클릭 어려움 회피). |
