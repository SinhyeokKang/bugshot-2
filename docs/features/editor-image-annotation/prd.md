# 에디터 삽입 이미지 어노테이션

## 배경

BugShot 리포트 본문(Tiptap 에디터)에는 drop·붙여넣기·파일 선택·인라인 영역 캡처로 이미지를 넣을 수 있다. 하지만 일단 에디터에 들어간 이미지는 **손댈 수 없다** — 화살표로 강조하거나 민감 정보를 가리려면 이미지를 지우고 밖에서 편집한 뒤 다시 넣어야 한다. 반면 캡처 스크린샷(미디어 섹션)에는 이미 Konva 기반 어노테이션 오버레이(`AnnotationOverlay`)가 붙어 있다. 같은 편집 능력을 본문 삽입 이미지에도 주는 것이 이 기능이다.

**왜 지금**: 스크린샷 어노테이션은 이미 있는데 본문에 붙인 이미지만 편집 불가라 UX가 비대칭이고, 특히 민감 정보 마스킹은 프라이버시 코어밸류와 직결된 흔한 요구다. 오버레이 캔버스가 이미 존재해 캡처 캔버스만 재사용하면 되므로 지금이 비용 대비 효율이 가장 좋다.

## 목표

- 에디터에 삽입된 인라인 이미지 위에 hover 시 우상단 버튼 그룹 `[초기화][어노테이션][삭제]`를 노출한다. 시각·상호작용은 코드블럭 collapse의 우상단 액션 그룹과 동일한 결(hover 노출, 아이콘 버튼, `block-actions` 스타일).
- **어노테이션**: 기존 `AnnotationOverlay`의 편집 캔버스를 재사용해 해당 이미지를 편집하고, 결과를 그 이미지 자리에 반영한다. (재사용은 캔버스에 국한 — 편집 결과를 담을 **이미지 NodeView와 N개 이미지용 원본 백업 store는 이 기능이 새로 도입**한다. 스크린샷은 단일쌍만 있어 그대로 쓸 수 없다. 상세는 design.md.)
- **초기화**: 어노테이션 이전 원본 이미지로 되돌린다. 어노테이션 기록이 있을 때만 노출한다.
- **삭제**: 별도 확인 없이 즉시 이미지 노드를 제거하고 연결된 blob을 정리한다.
- 작성 중(`DraftingPanel`)·제출 후 편집(`DraftEditDialog`) 양쪽 모두에서 동작한다(둘 다 `TiptapEditor`를 쓰므로 에디터 자체에 붙여 한 번에 커버).

## 비목표 (Non-goals)

- **도형 벡터 영속화·재편집 복원 없음.** 어노테이션 결과는 기존 캡처 어노테이션과 동일하게 래스터(webp)로만 남는다. 다시 어노테이션하면 현재 표시 중인(어노테이션된) 이미지를 배경으로 새로 그린다. 이전에 그린 화살표/박스를 개별 도형으로 되살리지 않는다.
- **부분 초기화 없음.** "초기화"는 어노테이션 전 원본으로의 전체 복귀만 제공한다. 도형 하나만 지우는 기능은 어노테이션 오버레이 편집 세션 내부의 undo/삭제로만 가능(기존 동작).
- 캡처 스크린샷 미디어 섹션의 기존 어노테이션 UI(`annotation-edit`/`annotation-remove`)는 변경하지 않는다. 이 기능은 **본문 삽입 이미지** 전용이며 별도 경로다.
- 삭제 확인 다이얼로그·undo·휴지통 없음. **어노테이션 작업물이 든 이미지도 무확인 즉시 삭제**한다(기존 삭제 관용구와 일관 — 첨부·env 행·코드블럭 전부 즉시). 오클릭 손실 리스크는 감수.
- **빈 어노테이션 Done은 no-op**이다. `AnnotationOverlay`는 도형 0개면 Done이 `onComplete`를 조기 리턴하므로, 아무것도 안 그리고 Done = Cancel과 동치(변경·백업 없음). 재어노테이션에서 "도형 없이 저장" 경로는 제공하지 않는다.
- **터치·키보드 도달 경로는 이번 스코프 밖**이다. 버튼은 hover(+`:focus-within`) 노출이라 no-hover(터치) 환경엔 대체 진입점이 없다. 사이드패널이 데스크톱 위주라 유예 — 필요 시 후속.

## 사용자 시나리오

### S1. 어노테이션 (최초)
1. 사용자가 본문 이미지에 마우스를 올린다 → 우상단에 `[어노테이션][삭제]` 버튼이 fade-in 된다(어노테이션 기록 없으므로 `[초기화]` 미노출).
2. `[어노테이션]` 클릭 → 풀스크린 `AnnotationOverlay`가 그 이미지를 배경으로 열린다.
3. 화살표·박스 등을 그리고 "Done" → 오버레이가 닫히고, 이미지가 어노테이션된 버전으로 갱신된다. (도형을 하나도 안 그리고 Done을 누르면 아무 변경 없이 닫힌다 — Cancel과 동치.)
4. 이후 hover 하면 `[초기화][어노테이션][삭제]` 3개가 모두 보인다.

### S2. 재어노테이션
1. 어노테이션된 이미지에서 `[어노테이션]` 다시 클릭 → 오버레이가 **현재 어노테이션된 이미지**를 배경으로 연다(도형은 빈 히스토리에서 시작).
2. Done → 다시 그 위에 구워진 새 이미지로 갱신. 원본은 여전히 백업에 보존된다.

### S3. 초기화
1. 어노테이션된 이미지에서 `[초기화]` 클릭 → 즉시 어노테이션 전 원본 이미지로 되돌아간다.
2. `[초기화]` 버튼이 사라진다(기록 없어짐).

### S4. 삭제
1. 이미지 hover → `[삭제]` 클릭 → 확인 없이 즉시 이미지가 본문에서 사라진다.
2. 원본·어노테이션 blob 모두 정리 대상이 된다.

### 엣지 케이스
- **작은 이미지**: 파비콘·아이콘 캡처(≈60px)처럼 버튼 그룹(~96px)보다 좁은 이미지에서도 그룹이 이미지 밖으로 삐져나오지 않는다 — 인라인 이미지에 `min-width`(버튼 그룹 폭 이상)를 강제해 최소 폭을 보장한다.
- **세션 복원**: 사이드패널을 닫았다 열어도(session/local 복원) 어노테이션된 이미지가 그대로 보이고, `[초기화]` 노출 여부(어노테이션 기록 유무)가 유지된다.
- **여러 이미지**: 한 본문에 이미지가 여러 개면 각 이미지의 어노테이션·원본·기록 상태가 독립적으로 관리된다.
- **제출/미리보기**: 최종 리포트로 나가는 이미지는 항상 "현재 표시 중"(어노테이션됐으면 어노테이션된) 버전이다. 원본 백업은 외부로 나가지 않는다.
- **오버레이 취소(Cancel)**: 어떤 변경도 반영하지 않고 닫는다(이미지 그대로).

## 성공 기준

- 본문 삽입 이미지 hover 시 우상단 버튼 그룹이 코드블럭과 동일한 스타일로 뜬다.
- 어노테이션 → 이미지가 그 자리에서 갱신되고, 최종 제출 이미지도 갱신된 버전이다.
- 초기화 → 어노테이션 전 원본으로 정확히 복귀한다. 어노테이션 기록이 없으면 버튼이 안 뜬다.
- 삭제 → 즉시 제거되고, 다음 prune 사이클에 **`inlineImages[refId]`와 원본 백업 `inlineImageOrigins[refId]`가 함께 orphan 정리**된다(원본 백업 store는 prune 대상 밖이지만 삭제로 markdown 참조가 사라지면 동일 orphan 집합으로 함께 회수). blob orphan 잔존 없음.
- 세션 닫았다 열어도 어노테이션 상태·기록 유무가 유지된다.
- `DraftingPanel`과 `DraftEditDialog` 양쪽에서 동일하게 동작한다.
- 기존 캡처 미디어 섹션 어노테이션·직렬화·orphan prune 회귀 없음.
