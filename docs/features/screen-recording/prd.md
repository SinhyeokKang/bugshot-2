# 화면 전체 녹화 (Screen Recording)

## 배경

현재 비디오 녹화는 `chrome.tabCapture`로 **현재 탭의 뷰포트만** 잡는다. 다음 한계가 있다:

- 브라우저 탭 바깥(다른 앱 창, OS 다이얼로그, 멀티 윈도우 플로우)을 보여줄 수 없다.
- 탭 경계를 넘나드는 재현(새 창 팝업, 외부 앱 연동)을 녹화로 전달할 방법이 없다.

QA·디자인 리뷰에서 "탭 밖에서 일어나는 일"을 이슈에 담으려면 화면/창 단위 녹화가 필요하다. 동시에, 현재 idle(이슈 작성 진입) 화면의 캡처 버튼이 늘면서 녹화 진입로의 위계가 흐려졌다 — 녹화 2종(뷰포트/화면)과 즉시성 도구(30s 리플레이)를 시각적으로 분리해야 한다.

## 목표

- `getDisplayMedia` 기반 **화면 전체 녹화** 모드를 추가한다. 사용자가 브라우저 네이티브 picker에서 [전체 화면 / 특정 앱 창 / 탭] 중 대상을 고른다.
- 결과물은 기존 뷰포트 녹화와 **동일한 경로**(drafting Media 섹션, IndexedDB `video:{tabId}`, 제출 시 첨부)로 흐른다 — 사용자에게 "또 다른 녹화 영상"으로 일관되게 보인다.
- idle 화면 캡처 버튼을 **1×2×2×1** 레이아웃으로 재배치한다:
  - Row1(1): 요소 선택
  - Row2(2): 요소 캡처 · 영역 스크린샷
  - Row3(2): 뷰포트 녹화 · 화면 전체 녹화
  - Row4(1): 30s 리플레이 (단독·하단)
  - footer는 기존대로 가이드 · 프리폼 2열 유지.

## 비목표 (Non-goals)

- **오디오 녹음** 미포함. 영상만(`audio: false`) — 기존 뷰포트 녹화와 동일.
- **새 단축키** 미추가. Chrome `suggested_key` 4개 상한이 이미 꽉 차 있다(패널토글 + element/screenshot/video). 화면 전체 녹화는 버튼으로만 진입.
- **로그 첨부 정책 변경** 없음. 화면 전체 녹화도 현재 탭의 console/network/action 로그를 그대로 첨부한다(뷰포트 녹화와 동일). 다른 창을 녹화하면 영상↔로그 탭이 다를 수 있으나, 사용자가 drafting에서 로그 섹션을 끌 수 있어 수용한다.
- **새 captureMode 추가** 없음. `captureMode: "video"`를 공유한다 — 소스(tab vs screen)만 다르고 결과 처리·UI는 동일.
- **desktopCapture / 화면 영역 크롭 / 녹화 중 주석** 미포함.

## 사용자 시나리오

### 주 플로우: 화면 전체 녹화
1. 사용자가 디버그 탭 > 이슈 작성 idle 화면에서 **[화면 전체 녹화]** 버튼을 클릭한다.
2. 브라우저 네이티브 화면 공유 picker가 즉시 뜬다. 사용자가 [전체 화면 / 창 / 탭] 중 하나를 고르고 "공유"를 누른다.
3. 녹화가 시작되고 패널은 recording 화면(기존 RecordingState — 경과 시간·중지 버튼)으로 전환된다.
4. 사용자가 재현 동작을 수행한다(탭 밖 동작 포함).
5. 다음 중 하나로 녹화가 끝난다:
   - 패널의 **중지** 버튼 클릭
   - 브라우저 화면 공유 막대의 **"공유 중지"** 클릭 (track ended)
   - 60초 최대 길이 도달 (자동 중지)
6. drafting phase로 전환되어 Media 섹션에 녹화 영상이 표시되고, 현재 탭의 로그가 첨부된다.

### 엣지 케이스
- **picker에서 취소**: 사용자가 공유 picker를 닫으면(`getDisplayMedia` reject) idle 상태 유지, 아무 일도 일어나지 않는다(no-op).
- **공유 중지 버튼**: 브라우저가 제공하는 "공유 중지"를 누르면 video track의 `ended` 이벤트로 녹화가 정상 종료되어 drafting으로 간다.
- **getDisplayMedia 미지원/거부**: 환경이 API를 막으면 콘솔 경고 후 idle 유지(뷰포트 녹화의 권한 만료 처리와 유사하게 graceful).
- **사이드패널 자체가 화면에 포함**: 전체 화면 선택 시 사이드패널 UI도 녹화될 수 있으나 기능상 무관.

## 성공 기준

- idle 화면이 1×2×2×1 레이아웃으로 렌더되고, [화면 전체 녹화] 버튼(`data-testid="mode-screen-record"`)이 뷰포트 녹화 옆에, 30s 리플레이가 하단 단독으로 배치된다.
- [화면 전체 녹화] 클릭 → getDisplayMedia picker → 대상 선택 → 녹화 → 중지 → drafting Media 섹션에 영상 표시 + 재생 가능.
- "공유 중지" / 패널 중지 / 60초 자동중지 세 경로 모두 drafting으로 정상 종료.
- picker 취소가 idle을 깨지 않는다.
- 기존 뷰포트 녹화·30s 리플레이·요소/스크린샷 캡처가 회귀 없이 동작.
- `docs/privacy.md`에 화면 전체 녹화(탭 밖 데이터 캡처 가능)가 반영되고 시행일이 갱신된다.
