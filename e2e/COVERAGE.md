# e2e 커버리지 맵 · 수동 잔여

## 수동 잔여 (자동화 못 한 것 — 이유 포함)

스크립트로 판정 불가하거나 e2e 환경 제약으로 빠진 항목. 새로 자동화하면 위 맵으로 옮긴다.

- **picker unsupported URL 중 webstore 차단 호스트** — 실제 webstore 접근이 필요해 fixture 서버로 재현 불가 (미지원 *스킴* 가드는 `unsupported-url.spec.ts`가 chrome://로 커버).
- **iframe 요소 캡처 크롭 정확도·top overlay 미포함** — before/after·요소 스크린샷의 크롭 좌표(offset 변환·topViewport 스케일)와 "top overlay가 캡처에 안 찍힘"은 `captureVisibleTab` 픽셀 판정이라 e2e 불가(+캡처 진입 spec은 quota flaky 정책상 금지 — GOTCHAS). 좌표 순수부는 `frame-geometry.test.ts`(composeTopRect·arm 게이트·스푸핑 방어) 단위로, 크롭 시각 정합은 수동(tasks.md).
- **iframe hover 하이라이트 시각 정합·blocker 핸드오프 경계 깜빡임·다수 iframe(광고) 성능** — overlay는 shadow DOM 시각 요소라 셀렉터 판정 불가(선택 성공이 핸드오프+클릭 라우팅을 함의 — `picker-iframe.spec`). 인접 링크 오네비게이션·경계 깜빡임·프레임 수십 개 성능 체감은 수동(tasks.md).
- **수동 캡처 video 모드**(탭 녹화 — `recordingMode="tab"`일 때의 `mode-record`) — `getUserMedia`+`chrome.tabCapture` 의존이라 headed 자동화에서 fake media 없이는 불안정.
- **konva 주석 — 도형 시각 정합·6도구 그리기/이동/리사이즈/회전·undo/redo·색상·두께·텍스트 인라인 입력** — Konva Stage가 단일 `<canvas>`라 도형이 DOM 노드가 아니어서 "무엇이 어떻게 그려졌나"를 셀렉터로 판정 불가(픽셀 비교 영역). `annotation-overlay.spec.ts`가 캔버스 *외부* 신호(오버레이 open/close·도구 active·img src 전이)만 커버하고, rect 1개 드래그로 done 활성화까지만 탄다. 순수 도형 로직(좌표 변환·undo/redo·empty 판정·transform 흡수)은 `shapes.test.ts`/`history.test.ts`/`presets.test.ts` 단위로 커버. 큰 스크린샷 자연 해상도 출력·좁은 폭 3단 툴바 리플로우·세션 복원(`screenshotAnnotated`)·IndexedDB before 첨부 반영은 수동(tasks.md).
- **녹화 중 그리기(펜/형광펜) — 실 녹화 합성·3초 페이드·그리기 툴바 노출·Esc 동기화·내비게이션 재표시** — `recording-annotation.spec`이 `annotation.show/setTool/hide` 메시지 직구동으로 오버레이 마운트·획 추가/미추가·pass-through·제거를 커버하지만, (a) 그린 획이 tabCapture/getDisplayMedia 스트림에 실제로 담기는지는 실 녹화(headed 자동화 불안정)라 수동, (b) 획이 ~3초 후 그린 순서대로 페이드아웃되는지는 타이머 시각 판정이라 수동, (c) RecordingState 그리기 툴바(색/펜·형광펜/두께) 노출·클릭→store→setTool 전송·crosshair는 실 녹화 진입(phase=recording)이 전제라 수동(오버레이가 받은 스타일을 획 path에 반영하는 부분은 `recording-annotation.spec`이 setTool 직구동으로 커버), (d) 페이지 Esc→툴 OFF 업스트림 동기화·녹화 중 내비게이션 재표시는 실 녹화+패널 조합이라 수동. 순수분(`pointsToPath`·`smoothPoint`·`overlayStrokeStyle`·`matchesOwnHost`·`annotationTool` 리셋)은 단위 테스트로 커버.
- **화면 전체 녹화**(화면 녹화 — `recordingMode="screen"`일 때의 `mode-record`) — `getDisplayMedia` 네이티브 picker가 자동화에서 안 떠(fake display 없이는 스트림 미획득) 클릭→녹화→drafting을 e2e로 못 탄다. **버튼 노출·1×2×2 DOM 구조·모드별 아이콘 전환은 `capture-modes-layout.spec.ts`가 커버**, 실제 녹화·source 라벨(탭/화면)·공유 중지·60초·picker 취소는 수동(tasks.md). getDisplayMedia가 side panel에서 동작하는지의 PoC도 수동 선행. idle 시각 정합(primary 강조·3행 정렬)·좁은 폭 라벨 truncate·리플레이 활성+인코딩 중 Row3 레이아웃도 수동.
- **단축키 4종**(`_execute_action`·capture-*) — Playwright는 확장 manifest commands를 발화할 수 없다(CDP 미지원).
- **permission-expired 다이얼로그** — activeTab 권한 만료 상태를 자동화로 재현하기 어렵다 (unsupported 분기만 커버).
- **광역 권한 보유 시 drafting cross-origin → deferred 만료 다이얼로그 부재** — 보존 세션 cross-origin은 keep(메시지 미발신)이라 "다이얼로그가 안 뜸"이라는 *부재* 단언이고, 가짜 패널 + deferred 메시지 broadcast는 SW로 관찰 불가. 분기 자체는 `resolveNavigationAction` 표 테스트(보존+커버→keep)와 `activetab-broad-permission.spec`의 비보존 keep 경로로 커버, 다이얼로그 부재는 tasks.md 수동 테스트로 검증.
- **광역 *미보유* 경로(닫힘·deferred)** — `<all_urls>`가 required host_permission으로 승격돼 광역 권한은 항상 보유 → 미보유 분기는 **프로덕션에 도달 불가**(과거 optional 시절의 경로). 순수함수 분기 자체는 `tab-bindings.test.ts`의 `legacyCases`(broadGranted=false)가 락인한다.
- **afterImage·캡처 썸네일 시각 정합** — 픽셀 비교라 스크립트 판정 불가. 행 수·색상 등으로 간접 단언만.
- **30s Replay 트리밍 전체** — 트림 오버레이는 `captureVisibleTab` 캡처로 진입해서 e2e(`replay-trim`·`replay-trim-logs`)가 환경 flaky로 **제거됨**. 로직은 단위 테스트로 커버한다: 초↔프레임 매핑·트림 경계(`trim-math` `secondsToFrameRange`/`frameOffsetsMs`/`previewTrimBounds`/`isTrimmedOut`), 가드밴드(`apply-trim`), 마커 환산·variant 필터(`buildErrorMarkers` positionPct·console error|warn / network 4xx·5xx·error·pending / action navigate). 오버레이 UI(아이콘 4탭·핸들 드래그/툴팁·muted=실제잘림 parity·재인코딩 영상 길이·에러 마커 위치·마커 클릭→탭 전환·로그 0건 empty·재생 게이팅)는 전부 **수동**(tasks.md).

- **cross-origin 스타일 보강(양성 경로) — specified author 값 채움·source 셀렉터 라벨·`var()` 해석·부분 보강 fallback·cross-origin-only 토큰 swatch hint** — background가 외부 cross-origin CSS를 fetch해 빈 specified를 채우는 양성 경로는 e2e로 재현 불가. e2e fixture 서버는 loopback 전용(127.0.0.1 top + localhost sheet로 origin 분리)인데, **SSRF 가드(`isFetchableSheetUrl`)가 localhost·127.x를 loopback으로 차단**(보안상 정상)해 보강 fetch가 안 일어난다. 가드를 통과하는 cross-origin 호스트(공개 도메인/IP)가 loopback 서버엔 없음. 가드 로직은 `ssrf-guard.test.ts`(35 케이스: 공개 통과 / loopback·link-local·RFC1918·IPv6 ULA·IPv4-mapped·FQDN 후행점 차단), 머지·인덱싱·var 참여는 `css-resolve.test.ts > mergeCrossOriginDecls`·`css-source-cache.test.ts > indexCrossOriginRules`, fallback 분기는 `sectionDefaultOpen.test.ts`(부분 보강 collapse·실패 시 computed 유지), stale 가드는 `editor-store.test.ts`로 커버. **cross-origin :root에만 정의된 custom prop이 토큰 목록에 들어가 swatch/hex hint가 뜨는 경로(`collectTokens`가 `getCrossOriginCustomProps()`를 same-origin 우선 merge)도 같은 SSRF 차단으로 e2e inert** — `getCrossOriginCustomProps()`가 `{}`라 `mergeCrossOriginTokens`가 no-op. 머지 헬퍼는 `css-resolve.test.ts > mergeCrossOriginTokens`(5 케이스: 추가·same-origin 우선·부분 충돌·빈 입력·비-- 방어)로 직접 커버. 실 보강(공개 CDN·naver)은 수동(tasks.md). **graceful fallback(가드 차단 → computed 유지, 에러 없음) 회귀는 `style-cross-origin-section.spec`이 새 코드에서 green으로 커버.**
