# BugShot 가이드

웹에서 버그를 발견할 때마다 "이거 어떻게 설명하지…" 하고 막막했던 적, 다들 있으시죠. BugShot은 바로 그 순간을 도와드립니다.

BugShot은 웹 페이지의 DOM 요소를 골라 스타일을 수정·비교한 뒤, Jira·GitHub·Linear·Notion·GitLab·Asana 이슈로 등록하는 Chrome 사이드패널 확장입니다. 버그를 발견한 그 화면에서 바로 캡처하고, 재현 환경·스크린샷·영상·로그를 한 번에 담아 리포트로 보냅니다.

"여기 이 버튼 색이 이상해요"를 말로 길게 설명하던 작업, 이제 클릭 한 번이면 충분합니다.

## 화면 구성: 메인 탭 4개

복잡해 보여도 걱정 마세요. 사이드패널은 딱 네 개의 탭으로 나뉘어 있어서, 하나씩 보면 금방 익숙해집니다.

- **디버그** — 캡처 모드(요소 선택·화면 캡처·영상 녹화)를 고르고 이슈를 작성하는 곳입니다. 콘솔·네트워크 로그도 여기서 봅니다.
- **이슈 목록** — 작성한 초안(Draft)과 제출한 이슈(Submitted)를 모아 보고, 언제든 다시 엽니다.
- **연동** — Jira·GitHub 같은 플랫폼을 연결합니다. 아직 연결이 없으면 이 탭으로 먼저 안내해 드립니다.
- **설정** — 이슈 본문 구성, AI 연결, 언어·테마를 원하는 대로 맞춥니다.
